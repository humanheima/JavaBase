# LeetCode 26. 删除有序数组中的重复项（Remove Duplicates from Sorted Array）解题文档

## 一、题目描述

给你一个 **非严格递增排列** 的数组 `nums`，请你 **原地** 删除重复出现的元素，使每个元素只出现一次，返回删除后数组的新长度 `k`。元素的相对顺序应保持一致。

**判题标准：** 假设返回了 `k`，那么 `nums` 的 **前 `k` 个元素** 应包含去重后的全部元素且顺序与原数组一致；`nums` 在 `k` 之后的元素无所谓。

**示例：**

| 输入 | 输出 k | 前 k 项 | 说明 |
|------|--------|---------|------|
| `[1,1,2]` | `2` | `[1,2]` | 删掉一个重复的 1 |
| `[0,0,1,1,1,2,2,3,3,4]` | `5` | `[0,1,2,3,4]` | 每个值保留一个 |
| `[1]` | `1` | `[1]` | 单元素 |
| `[2,2,2,2]` | `1` | `[2]` | 全相同只剩一个 |

**链接：** https://leetcode.cn/problems/remove-duplicates-from-sorted-array/

---

## 二、核心观察

1. **数组已排序是关键前提**：相同的元素一定 **相邻**，因此判断"是否重复"只需比较当前元素和**前一个已保留的元素**，无需哈希表或全局比较。
2. **原地 + O(1) 空间要求**：不能新开数组，要在原数组上覆盖写入 → 用 **双指针（快慢指针）** 把去重后的元素依次搬到数组前部。
3. **返回的是长度，不是数组**：`k` 之后的内容不重要，只要前 `k` 个正确即可。
4. **下标与长度的换算**：慢指针 `slow` 指向"已去重区间的末尾下标"，最终长度 = `slow + 1`。

---

## 三、解法

### 解法：快慢指针（双指针，唯一推荐）

**思路：**

1. `slow` 指向已去重区间的最后一个位置，初始为 `0`（第一个元素必然保留）。
2. `fast` 从下标 `1` 开始向后扫描每个元素。
3. 比较 `nums[fast]` 与 `nums[slow]`：
   - **相等**：说明是重复值，`fast` 直接跳过；
   - **不等**：遇到新值，`slow++` 后把 `nums[fast]` 写到 `nums[slow]`。
4. 扫描结束后，区间 `[0, slow]` 就是去重结果，长度为 `slow + 1`。

**过程演示**（`[0,0,1,1,2]`）：

```
slow=0                       [0, 0, 1, 1, 2]
fast=1: nums[1]=0 == nums[0]=0 → 跳过
fast=2: nums[2]=1 != nums[0]=0 → slow=1, nums[1]=1   [0, 1, 1, 1, 2]
fast=3: nums[3]=1 == nums[1]=1 → 跳过
fast=4: nums[4]=2 != nums[1]=1 → slow=2, nums[2]=2   [0, 1, 2, 1, 2]
结束：长度 = slow+1 = 3，前 3 项 [0,1,2]
```

**关键代码：**

```java
public int removeDuplicates(int[] nums) {
    if (nums == null || nums.length == 0) return 0;
    int slow = 0; // [0..slow] 是已去重的部分
    for (int fast = 1; fast < nums.length; fast++) {
        if (nums[fast] != nums[slow]) {
            slow++;
            nums[slow] = nums[fast];
        }
    }
    return slow + 1; // 长度 = 末尾下标 + 1
}
```

- **时间复杂度：** `O(n)` —— `fast` 扫描数组一遍。
- **空间复杂度：** `O(1)` —— 只用两个指针，原地覆盖。
- **优点：** 一次遍历、原地、空间最省，是本题标准解。
- **缺点：** 仅适用于 **已排序** 数组；无序数组需另想办法（排序或哈希）。

---

## 四、易错点总结

1. **比较对象是 `nums[slow]` 而非 `nums[fast-1]`**：虽然在本题（保留 1 个）两者等价，但用 `nums[slow]`（已保留区间的末尾）语义更清晰，也便于推广到"最多保留 k 个"的变体（LeetCode 80）。
2. **先 `slow++` 再赋值**：顺序不能反，否则会覆盖掉已保留的元素。
3. **返回长度 = `slow + 1`**：`slow` 是下标，长度要 +1，别直接返回 `slow`。
4. **空数组边界**：`length == 0` 要返回 `0`，否则 `slow+1` 会错误返回 1。
5. **依赖有序前提**：本解法只对有序数组成立；若数组无序，相同元素不相邻，会去重失败。
6. **是删元素不是删值**：要求"原地"，不能 new 新数组返回。
