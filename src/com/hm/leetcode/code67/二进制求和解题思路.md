# LeetCode 67. 二进制求和（Add Binary）解题文档

## 一、题目描述

给你两个二进制字符串 `a` 和 `b`，以二进制字符串的形式返回它们的和。

**示例：**

| a | b | 输出 | 说明 |
|---|----|------|------|
| `"11"` | `"1"` | `"100"` | 3 + 1 = 4，二进制 100 |
| `"1010"` | `"1011"` | `"10101"` | 10 + 11 = 21，二进制 10101 |

**提示：**
- `1 <= a.length, b.length <= 10^4`
- `a` 和 `b` 仅由字符 `'0'` 或 `'1'` 组成
- 字符串如果不是 `"0"`，就不含前导零

**链接：** https://leetcode.cn/problems/add-binary/

---

## 二、核心观察

1. **二进制加法与十进制加法本质相同**：都是从最低位开始，逐位相加并处理进位。不同之处仅在于进位规则（二进制是满 2 进 1）。
2. **字符串长度可能不同**：需要处理较短字符串已经处理完的情况，此时可视为该位为 0。
3. **不能直接使用整数类型**：两个字符串最长可达 10^4 位，远超 `long` 的表示范围，必须模拟逐位计算。
4. **进位只有 0 或 1**：这是二进制的特点，简化了判断逻辑（`carry = sum / 2`）。
5. **结果构建顺序**：从低位到高位计算得到的是逆序结果，最终需要反转字符串。
6. **最高位可能产生新进位**：当两个字符串最高位相加后仍有进位时，结果长度会比原输入更长 1 位（如 `"1"+"1"` → `"10"`）。

---

## 三、解法

### 解法：双指针从后往前模拟加法（推荐）

**思路：**

1. 使用两个指针 `aIndex` 和 `bIndex` 分别指向字符串 `a` 和 `b` 的末尾（最低位）。
2. 初始化 `carry = 0` 用于记录进位。
3. 使用 `StringBuilder` 收集结果（从低位到高位收集，之后反转）。
4. 循环条件：只要还有任意一个字符串还有位未处理（`aIndex >= 0 || bIndex >= 0`）。
5. 每次循环：
   - 取出当前位数字（若指针已越界则取 0）：`num1 = a.charAt(aIndex) - '0'`（或 0）。
   - 计算当前位总和 `sum = num1 + num2 + carry`。
   - 当前位结果为 `sum % 2`，进位为 `sum / 2`。
   - 将结果位 `append` 到 StringBuilder。
   - 两个指针同时左移。
6. 循环结束后，如果 `carry == 1`，说明最高位还有进位，需要额外 append 1。
7. 最后将 StringBuilder 反转并转为字符串返回。

**关键代码：**

```java
public String addBinary(String a, String b) {
    int aIndex = a.length() - 1;
    int bIndex = b.length() - 1;

    int carry = 0;
    StringBuilder sb = new StringBuilder();

    while (aIndex >= 0 || bIndex >= 0) {
        int num1 = aIndex >= 0 ? a.charAt(aIndex) - '0' : 0;
        int num2 = bIndex >= 0 ? b.charAt(bIndex) - '0' : 0;
        int sum = num1 + num2 + carry;
        carry = sum / 2;
        sb.append(sum % 2);
        aIndex--;
        bIndex--;
    }

    if (carry == 1) {
        sb.append(1);
    }

    return sb.reverse().toString();
}
```

- **时间复杂度：** `O(max(len(a), len(b)))`，每个字符最多处理一次。
- **空间复杂度：** `O(max(len(a), len(b)))`，主要用于存储结果的 StringBuilder（反转后返回的字符串也需要空间）。
- **优点：** 
  - 逻辑清晰，直接模拟小学竖式加法。
  - 能正确处理长度不等、最高位进位等所有情况。
  - 仅使用常数额外变量，代码简洁。
- **缺点：** 需要最后一次 reverse 操作（可通过从后往前填充数组避免，但复杂度相当）。

---

### 其他思路简述

- **使用 BigInteger**：`new BigInteger(a, 2).add(new BigInteger(b, 2)).toString(2)` —— 代码最短，但面试中通常不被接受（隐藏了模拟过程）。
- **从前向后处理（较少见）**：需要先对齐长度、预先判断是否会产生新最高位，实现更繁琐。
- **位运算优化**：在某些语言中可用 `^`（异或）和 `&`（与）模拟加法，但 Java 中对字符串的逐位处理优势不大。

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 | 推荐场景 |
|------|----------|------|------|------|------|----------|

## 五、易错点总结

1. **指针从末尾开始**：容易写成从 0 开始，导致高低位搞反。
2. **长度不等时的 0 补齐**：必须在取值时判断指针是否越界，否则会抛出 `StringIndexOutOfBoundsException`。
3. **进位计算顺序**：先计算 `sum % 2` 得到当前位，再用 `sum / 2` 更新 carry。顺序反了会导致错误。
4. **最后处理剩余进位**：很多人在循环结束后忘记判断 `if (carry == 1)` 额外追加 1。
5. **反转时机**：必须在所有位和最终进位都 append 完之后再 `reverse()`，中途反转会出错。
6. **字符转数字**：使用 `charAt(i) - '0'` 是最常见且高效的写法，不要用 `Integer.parseInt(String.valueOf(...))`。
7. **StringBuilder 的 append 顺序**：从低位 append，高位在后面，最后统一 reverse 是最自然的写法。
8. **返回类型是 String**：不要返回 int 或其他类型，题目明确要求二进制字符串形式。
