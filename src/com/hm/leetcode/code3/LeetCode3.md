# LeetCode 3. 无重复字符的最长子串

> https://leetcode.cn/problems/longest-substring-without-repeating-characters/

## 一、问题描述

给定一个字符串 `s`，请你找出其中不含有重复字符的**最长子串**的长度。

注意：**子串**是连续的，区别于子序列。

| 输入 | 输出 | 解释 |
| --- | --- | --- |
| `"abcabcbb"` | 3 | 最长子串是 `"abc"` |
| `"bbbbb"` | 1 | 最长子串是 `"b"` |
| `"pwwkew"` | 3 | 最长子串是 `"wke"`（`"pwke"` 是子序列，不算） |
| `""` | 0 | 空串 |

## 二、核心思想：滑动窗口

暴力解法是枚举所有子串再判断是否有重复，时间复杂度 `O(n³)` 或 `O(n²)`，效率太低。

观察一个关键性质：

> 如果子串 `[left, right]` 内没有重复字符，那么当 `right` 向右移动遇到一个**新字符**时，
> 我们只需要判断这个新字符是否破坏了"无重复"这个条件，而不需要从头检查整个子串。

于是用一个**窗口** `[left, right]` 来维护"当前没有重复字符的子串"：

- `right` 指针：不断向右扩张，尝试把更多字符纳入窗口。
- `left` 指针：当窗口内出现重复字符时，向右收缩，把重复字符挤出窗口。

整个过程中，两个指针都只会向右移动，所以时间复杂度是 `O(n)`。

## 三、如何快速判断重复 & 收缩窗口

用一个 `HashMap<Character, Integer>` 记录**每个字符最近一次出现的下标**。

当 `right` 指向字符 `c` 时：

1. 如果 `c` 之前出现过，且它上次出现的位置 **在当前窗口内**，说明产生了重复。
   此时把 `left` 跳到 `c` 上次出现位置的**下一格**，一步到位把重复字符移出窗口。
2. 更新 `c` 的最新下标为 `right`。
3. 此时窗口 `[left, right]` 必然无重复，用 `right - left + 1` 更新最大长度。

### 为什么 left 要取 max？

```java
// 正确写法：取 max
left = Math.max(left, lastIndex.get(c) + 1);

// 错误写法：直接赋值
left = lastIndex.get(c) + 1;
```

#### 1. 两种写法的语义区别

`lastIndex.get(c)` 返回的是字符 `c` **历史上最后一次出现的下标**——注意它记录的是"全局历史"，
并**不保证**这个下标还落在当前窗口 `[left, right]` 内。

- **直接赋值**：不管 `c` 上次出现在哪，都强行把 `left` 跳到那个位置的下一格。
  这隐含了一个假设——"`c` 上次出现的位置一定在窗口内"。可这个假设并不总成立。
- **取 max**：只有当 `c` 上次出现的位置 **在当前 left 右边**（即仍在窗口内）时，才会推动 `left`；
  如果 `c` 上次出现的位置已经在 `left` 左边（早被挤出窗口了），`max` 会保留原来的 `left`，**不让它后退**。

一句话：**`left` 这个指针只能向右走，绝不能向左回退**。直接赋值会破坏这个不变量。

#### 2. 用 `"abba"` 看清楚"回退"是怎么发生的

逐步跟踪两种写法（窗口是 `[left, right]`，长度 = `right - left + 1`）：

| right | c | lastIndex 命中情况 | 直接赋值的 left | 取 max 的 left（正确） |
| --- | --- | --- | --- | --- |
| 0 | a | 无记录 | 0 | 0 |
| 1 | b | 无记录 | 0 | 0 |
| 2 | b | b 上次在下标 1，**仍在窗口内** | 1+1 = **2** | max(0, 2) = **2** |
| 3 | a | a 上次在下标 0，**已在窗口外** | 0+1 = **1** ❌ | max(2, 1) = **2** ✅ |

关键看 `right = 3`（第二个 `a`）这一步：

- 此时窗口已经是 `[2, 2]`（只含下标 2 的 `b`），`left = 2`。
- `a` 上次出现在下标 **0**，但下标 0、1 早就被挤出窗口了，**当前窗口里根本没有 `a`**，并不存在重复。
- **直接赋值**：`left = 0 + 1 = 1`。`left` 从 2 **倒退回了 1**！窗口变成 `[1, 3]` = `"bba"`，
  反而把已经移出去的 `b` 又圈了回来——这下窗口里既有下标 1 的 `b` 又有下标 2 的 `b`，
  **窗口内部出现了重复字符，"无重复"这个核心不变量被破坏了**。此时算出的长度 `3 - 1 + 1 = 3` 是错的。
- **取 max**：`left = max(2, 1) = 2`，`left` 保持不动，窗口是 `[2, 3]` = `"ba"`，长度 2，正确。

最终 `"abba"` 的正确答案是 **2**（`"ab"` 或 `"ba"`），直接赋值会错误地得到 **3**。

#### 3. 根因总结

`HashMap` 存的是"全局最后下标"，是一份**可能过期**的信息。当某个字符上次出现的位置已经滑到窗口左边以外时，
它对当前窗口已经不构成重复约束了。`Math.max(left, ...)` 的作用就是**过滤掉这种过期信息**，
保证 `left` 单调不减、窗口只向右滑动。

> 如果不想用 max，另一种思路是收缩窗口时把移出的字符从哈希结构里删掉（比如改用 `HashSet` + while 收缩），
> 这样哈希里永远只剩窗口内的字符，就不存在"过期下标"的问题了。但用 `HashMap` 记下标 + `max` 跳跃的写法更简洁、效率更高。

## 四、执行流程示例（"pwwkew"）

| right | c | left（更新后） | 窗口 | 长度 | maxLength |
| --- | --- | --- | --- | --- | --- |
| 0 | p | 0 | `p` | 1 | 1 |
| 1 | w | 0 | `pw` | 2 | 2 |
| 2 | w | 2 | `w` | 1 | 2 |
| 3 | k | 2 | `wk` | 2 | 2 |
| 4 | e | 2 | `wke` | 3 | 3 |
| 5 | w | 3 | `kew` | 3 | 3 |

最终答案 **3**。

## 五、代码

```java
public int lengthOfLongestSubstring(String s) {
    if (s == null || s.isEmpty()) {
        return 0;
    }

    Map<Character, Integer> lastIndex = new HashMap<>();
    int maxLength = 0;
    int left = 0;

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (lastIndex.containsKey(c)) {
            left = Math.max(left, lastIndex.get(c) + 1);
        }
        lastIndex.put(c, right);
        maxLength = Math.max(maxLength, right - left + 1);
    }

    return maxLength;
}
```

## 六、复杂度分析

- **时间复杂度** `O(n)`：`right` 遍历一遍字符串，`left` 总共也只向右移动 `n` 步，每个字符进出窗口各一次。
- **空间复杂度** `O(min(n, m))`：哈希表最多存放窗口内的字符，`m` 为字符集大小（如 ASCII 为 128）。

## 七、小结

滑动窗口的通用套路：

1. 右指针扩张，纳入新元素；
2. 当窗口不满足条件时，移动左指针收缩，直到重新满足条件；
3. 在满足条件的每一步更新答案。

本题的关键在于用 **HashMap 记录字符下标** 实现 `left` 的"一步跳跃"，并通过 `Math.max` 保证窗口不回退。
