# LeetCode 94. 二叉树的中序遍历（Binary Tree Inorder Traversal）解题文档

## 一、题目描述

给定一个二叉树的根节点 `root`，返回它的**中序遍历**结果。

中序遍历的顺序是 **左子树 → 根 → 右子树**（左 - 根 - 右）。

**示例：**

```
        1
       / \
      2   3
     / \   \
    4   5   6
```

| 输入 | 输出 | 说明 |
|------|------|------|
| 上图的 `root` | `[4, 2, 5, 1, 3, 6]` | 左 4 → 根 2 → 右 5 → 根 1 → 右子树(3 → 6) |
| `[]`（空树） | `[]` | 空树返回空列表 |
| `[1]` | `[1]` | 单节点 |

**链接：** https://leetcode.cn/problems/binary-tree-inorder-traversal/

---

## 二、核心观察

1. 中序遍历的本质是「**左 - 根 - 右**」：对任意节点，必须先访问完它的整棵左子树，才访问它自己，最后才访问右子树。
2. **递归**天然贴合这个定义，但有系统调用栈开销；树退化成链时栈深可达 O(n)。
3. **迭代**用一个显式栈模拟递归：不断把左链压栈，弹出即「根」，再转向右子树。空间仍是 O(n)。
4. **Morris 遍历**是进阶：利用叶子节点空闲的 `right` 指针建立指向「中序后继」的临时线索，把空间降到 **O(1)**，遍历后再拆除线索还原树。

---

## 三、三种解法

### 解法一：递归（最直观）

**思路：**
1. 递归边界：节点为空直接返回。
2. 先递归遍历左子树 → 再把当前节点值加入结果 → 再递归遍历右子树。

```java
private void inorder(TreeNode node) {
    if (node == null) {
        return;
    }
    inorder(node.left);   // 遍历左子树
    result.add(node.val); // 访问根节点
    inorder(node.right);  // 遍历右子树
}
```

- **时间复杂度：** O(n)，每个节点访问一次
- **空间复杂度：** O(n)，递归调用栈，最坏（链状树）为 O(n)，平衡时为 O(log n)
- **优点：** 代码最短，与中序定义一一对应
- **缺点：** 依赖系统栈，深树有栈溢出风险；用了共享成员 `result`，需在入口 `clear()`

---

### 解法二：迭代 + 显式栈

**思路：**
1. 用 `stack` 模拟递归，`current` 指向当前处理的节点。
2. **内层 while**：把 `current` 及其所有左子节点一路压栈，直到左子为空——栈顶即当前子树最左节点。
3. 弹出栈顶 = 「根」，记录其值。
4. 转向它的右子树：`current = current.right`，回到步骤 2。
5. 当 `current` 为空且栈也为空，遍历结束。

```java
public List<Integer> inorderTraversal2(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;

    Stack<TreeNode> stack = new Stack<>();
    TreeNode current = root;

    while (current != null || !stack.isEmpty()) {
        while (current != null) {     // 一路压入左链
            stack.push(current);
            current = current.left;
        }
        current = stack.pop();        // 弹出即“根”
        result.add(current.val);
        current = current.right;      // 转向右子树
    }
    return result;
}
```

- **时间复杂度：** O(n)
- **空间复杂度：** O(n)，显式栈最坏存 n 个节点
- **优点：** 不依赖系统栈，可控；面试常考
- **缺点：** 仍需 O(n) 辅助空间；循环条件 `current != null || !stack.isEmpty()` 易写错

---

### 解法三：Morris 遍历（O(1) 空间，进阶）

**思路：**
利用叶子节点空闲的 `right` 指针建立「线索」，省掉栈。对每个 `current`：
1. **若无左子树**：直接访问 `current`，转向右子树（可能是真右子，也可能是线索后继）。
2. **若有左子树**：找到左子树的**最右节点** `predecessor`（即 `current` 的中序前驱）：
   - 若 `predecessor.right == null`（第一次到达）：建立线索 `predecessor.right = current`，进入左子树。
   - 若 `predecessor.right == current`（第二次到达，说明左子树已遍历完）：**拆除线索**，访问 `current`，转向右子树。

```java
public List<Integer> inorderTraversal3(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    TreeNode current = root;

    while (current != null) {
        if (current.left == null) {
            result.add(current.val);          // 无左子树，直接访问
            current = current.right;
        } else {
            TreeNode predecessor = current.left;   // 找中序前驱
            while (predecessor.right != null && predecessor.right != current) {
                predecessor = predecessor.right;
            }
            if (predecessor.right == null) {  // 第一次：建立线索
                predecessor.right = current;
                current = current.left;
            } else {                          // 第二次：拆线索 + 访问
                predecessor.right = null;
                result.add(current.val);
                current = current.right;
            }
        }
    }
    return result;
}
```

- **时间复杂度：** O(n)，每条边最多被走 2 次（建线索、拆线索），总体仍线性
- **空间复杂度：** O(1)，不用栈也不用递归，仅几个指针
- **优点：** 唯一的常数空间解法，进阶亮点
- **缺点：** 逻辑最复杂；会临时修改树结构（虽最终还原），多线程环境下不安全

## 四、三种解法对比

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、递归 | 左-根-右 直接递归 | O(n) | O(n) 栈 | 最直观、最短 | 依赖系统栈，深树溢出 |
| 二、迭代 + 栈 | 显式栈压左链 | O(n) | O(n) | 可控、面试常考 | 仍需 O(n) 空间 |
| 三、Morris | 线索化 + O(1) 指针 | O(n) | O(1) | 常数空间 | 逻辑复杂、临时改树 |

---

## 五、易错点总结

1. **顺序别搞反**：中序是「左 - 根 - 右」。前序是「根 - 左 - 右」，后序是「左 - 右 - 根」，三者只是 `result.add` 的位置不同。
2. **递归共享 `result` 要清空**：本实现 `result` 是成员变量，主函数入口必须 `result.clear()`，否则多次调用结果会累积。
3. **迭代循环条件**：必须是 `current != null || !stack.isEmpty()`，两者用「或」；只判栈非空会在初始时直接退出。
4. **Morris 找前驱的内层条件**：`predecessor.right != null && predecessor.right != current` 缺一不可——前者防空指针，后者用于识别「线索已建立」从而终止查找。
5. **Morris 必须拆线索**：第二次到达时若忘记 `predecessor.right = null`，树结构被破坏，且可能死循环。
