# LeetCode 22. 括号生成（Generate Parentheses）解题文档

## 一、题目描述

数字 `n` 代表生成括号的对数，请你设计一个函数，生成所有可能的、并且 **有效的** 括号组合。

**有效括号组合**需满足：

- 恰好包含 `n` 个左括号 `(` 和 `n` 个右括号 `)`；
- 任意前缀中，左括号数量都不少于右括号数量（即括号能正确匹配）。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `n = 1` | `["()"]` | 只有一种 |
| `n = 2` | `["(())","()()"]` | 共 2 种 |
| `n = 3` | `["((()))","(()())","(())()","()(())","()()()"]` | 共 5 种（卡塔兰数 C₃） |
| `n = 0` | `[""]`（本实现返回 `[]`） | 约定 `n ≥ 1` 时无歧义 |

**链接：** https://leetcode.cn/problems/generate-parentheses/

---

## 二、核心观察

1. **结果数量是卡塔兰数**：`n` 对括号的有效组合数为 `C_n = (1/(n+1)) · C(2n, n)`，约等于 `4^n / √n`。这说明结果集本身就是指数级，任何算法都至少要 `O(结果数 × n)` 的时间。
2. **「左括号随时可加，右括号受限」是有效性的关键**：在构造过程中只要满足两条约束，结果必然有效，从而无需事后校验：
   - 已用左括号数 `< n` 时，才能加 `(`；
   - 已用右括号数 `<` 已用左括号数时，才能加 `)`（保证任意前缀左 ≥ 右）。
3. **本质是「带剪枝的决策树遍历」**：每一位有「放左 / 放右」两种选择，配合上面的约束剪掉所有无效分支，回溯法天然契合。

---

## 三、解法

### 解法一：回溯法（剩余计数版，推荐）

**思路：**

1. 用 `left`、`right` 分别表示**还剩多少个**左、右括号可用，初始都为 `n`。
2. 用一个 `StringBuilder current` 增量构建当前路径。
3. 递归中做选择：
   - 若 `left > 0`，可放 `(`，递归后回溯（删掉刚加的字符）；
   - 若 `right > left`（剩余右括号多于剩余左括号，等价于「已放左括号多于已放右括号」），可放 `)`，递归后回溯。
4. 当 `left == 0 && right == 0`，说明括号用完，`current` 是一个完整有效组合，收集结果。

> 为什么 `right > left` 能保证有效？剩余量 `left ≤ right` 恒成立（左括号总是先于配对的右括号放入）。`right > left` 即「剩余右 > 剩余左」，意味着已经放下的左括号比右括号多，此时补一个右括号才不会出现 `)` 多于 `(` 的非法前缀。

**关键代码：**

```java
public List<String> generateParenthesis(int n) {
    List<String> result = new ArrayList<>();
    backtrack(new StringBuilder(), n, n, result);
    return result;
}

/**
 * @param current 当前已构建的括号串
 * @param left    左括号剩余数量
 * @param right   右括号剩余数量
 */
private void backtrack(StringBuilder current, int left, int right, List<String> result) {
    // 终止条件：左右括号都用完
    if (left == 0 && right == 0) {
        result.add(current.toString());
        return;
    }
    // 添加左括号
    if (left > 0) {
        current.append('(');
        backtrack(current, left - 1, right, result);
        current.deleteCharAt(current.length() - 1); // 回溯
    }
    // 添加右括号（需确保右括号不多于左括号）
    if (right > left) {
        current.append(')');
        backtrack(current, left, right - 1, result);
        current.deleteCharAt(current.length() - 1); // 回溯
    }
}
```

- **时间复杂度：** `O(4^n / √n)` —— 结果有效组合数为卡塔兰数级别，每条合法路径长度 `2n`，剪枝后几乎不走无效分支。
- **空间复杂度：** `O(n)` —— 递归栈深度 `2n`，`StringBuilder` 长度 `2n`（不计输出本身）。
- **优点：** 剪枝彻底、不生成无效串、代码简洁、空间最省。
- **缺点：** 需要理解回溯与「剩余量约束」的等价转化。

### 解法二：回溯法（已用计数版，等价写法）

**思路：** 与解法一完全等价，只是把「剩余量」换成「已用量」`open`、`close`，更贴近直觉。

1. `open` = 已放左括号数，`close` = 已放右括号数。
2. 当 `current.length() == 2n` 时收集结果。
3. `open < n` 时可加 `(`；`close < open` 时可加 `)`。

**关键代码：**

```java
private void backtrack(StringBuilder cur, int open, int close, int n, List<String> res) {
    if (cur.length() == 2 * n) {
        res.add(cur.toString());
        return;
    }
    if (open < n) {
        cur.append('(');
        backtrack(cur, open + 1, close, n, res);
        cur.deleteCharAt(cur.length() - 1);
    }
    if (close < open) {
        cur.append(')');
        backtrack(cur, open, close + 1, n, res);
        cur.deleteCharAt(cur.length() - 1);
    }
}
```

- **时间复杂度：** `O(4^n / √n)`
- **空间复杂度：** `O(n)`
- **优点：** 约束条件 `open < n`、`close < open` 含义直白，初学者更易理解。
- **缺点：** 与解法一本质相同，无额外优势。

---

### 解法三：BFS / 队列法（仅作对比）

**思路：** 用队列保存中间状态 `(当前串, 已用左, 已用右)`，每次出队后按同样的约束扩展为新状态入队，直到串长达到 `2n` 收集结果。

- **时间复杂度：** `O(4^n / √n)`
- **空间复杂度：** `O(4^n / √n)` —— 队列要同时存大量中间状态。
- **优点：** 非递归，避免深递归栈。
- **缺点：** 空间开销显著大于回溯，代码更繁琐，实践中不推荐。

## 四、解法对比表

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、回溯（剩余计数） | 剩余左右量约束剪枝 | O(4ⁿ/√n) | O(n) | 简洁、省空间 | 需理解剩余量约束 |
| 二、回溯（已用计数） | 已用左右量约束剪枝 | O(4ⁿ/√n) | O(n) | 约束含义直白 | 与解法一等价 |
| 三、BFS/队列 | 队列逐层扩展状态 | O(4ⁿ/√n) | O(4ⁿ/√n) | 非递归 | 空间大、代码繁 |

**推荐：** 解法一/二的回溯法，二者本质相同，按个人习惯选「剩余」或「已用」计数即可。

---

## 五、易错点总结

1. **右括号约束写错**：必须是 `right > left`（剩余）或 `close < open`（已用），而不是 `right > 0`。否则会生成 `)(`、`())(` 等非法串。
2. **忘记回溯**：递归返回后必须 `deleteCharAt` 撤销本次选择，否则 `StringBuilder` 被污染，结果全错。共用同一个 `StringBuilder` 时尤其要注意。
3. **终止条件**：用 `left == 0 && right == 0`（剩余版）或 `length == 2n`（已用版），不要只判其一。
4. **n = 0 边界**：本实现会直接命中终止条件返回 `[""]` / `[]`，题目通常约定 `n ≥ 1`，注意按题面要求处理空串语义。
5. **字符串构建**：用 `StringBuilder` 增量构建 + 回溯，避免每层 `String` 拼接产生大量临时对象。
