# LeetCode 70. 爬楼梯（Climbing Stairs）解题文档

## 一、题目描述

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。

每次你可以爬 **1** 或 **2** 个台阶。你有多少种不同的方法可以爬到楼顶呢？

**示例：**

| n | 输出 | 说明 |
|---|------|------|
| 2 | 2 | (1+1), (2) |
| 3 | 3 | (1+1+1), (1+2), (2+1) |

**提示：**
- `1 <= n <= 45`

**链接：** https://leetcode.cn/problems/climbing-stairs/

---

## 二、核心观察

1. **状态转移方程**：到达第 `i` 阶的方法数 = 到达 `i-1` 阶的方法数 + 到达 `i-2` 阶的方法数。
   - 最后一步要么是「从 `i-1` 跨 1 阶上来」，要么是「从 `i-2` 跨 2 阶上来」。
   - 因此 `f(i) = f(i-1) + f(i-2)`，这是**斐波那契数列**的经典形式。

2. **边界条件**：
   - `f(1) = 1`（只有一种方式：跨 1 阶）
   - `f(2) = 2`（1+1 或 2）

3. **不需要记录完整路径**：题目只问「有多少种方法」，属于典型的计数型 DP，不需要回溯或记录具体方案。

4. **空间优化空间巨大**：由于每次只依赖前两个状态，完全可以用两个变量滚动更新，空间从 O(n) 降到 O(1)。

5. **n 最大 45**：`f(45)` 约为 18 亿，仍在 `int` 范围内（Java `int` 最大约 21 亿），无需使用 `long`。

---

## 三、解法

### 解法一：动态规划（滚动变量，推荐，O(1) 空间）

**思路：**

1. 如果 `n <= 2`，直接返回 `n`（边界情况）。
2. 初始化：
   - `prev2 = 1`（对应 `f(i-2)`）
   - `prev1 = 2`（对应 `f(i-1)`）
3. 从第 3 阶开始迭代到 `n`：
   - `curr = prev2 + prev1`
   - 更新 `prev2 = prev1`，`prev1 = curr`
4. 最终 `prev1`（或 `curr`）即为 `f(n)`。

**关键代码：**

```java
public int climbStairs(int n) {
    if (n <= 2) {
        return n;
    }

    int prev2 = 1;   // f(i-2)
    int prev1 = 2;   // f(i-1)
    int curr = 0;

    for (int i = 3; i <= n; i++) {
        curr = prev2 + prev1;
        prev2 = prev1;
        prev1 = curr;
    }
    return curr;
}
```

- **时间复杂度：** `O(n)`
- **空间复杂度：** `O(1)`
- **优点：** 空间最优、代码简洁、速度快，是本题最优解。
- **缺点：** 需要理解「只依赖前两个状态」才能进行空间优化。

---

### 解法二：动态规划（完整数组，教学版）

**思路：** 用一个数组 `dp` 显式存储所有子问题的结果。

```java
public int climbStairsDP(int n) {
    if (n <= 2) return n;
    int[] dp = new int[n + 1];
    dp[1] = 1;
    dp[2] = 2;
    for (int i = 3; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}
```

- **时间复杂度：** `O(n)`
- **空间复杂度：** `O(n)`
- **优点：** 状态转移最直观，易于理解和调试。
- **缺点：** 浪费空间。

---

### 解法三：记忆化搜索（自顶向下）

使用递归 + 备忘录，避免重复计算。

```java
private int[] memo;

public int climbStairsMemo(int n) {
    memo = new int[n + 1];
    return dfs(n);
}

private int dfs(int i) {
    if (i <= 2) return i;
    if (memo[i] != 0) return memo[i];
    return memo[i] = dfs(i - 1) + dfs(i - 2);
}
```

- **时间复杂度：** `O(n)`
- **空间复杂度：** `O(n)`（递归栈 + memo 数组）
- **优点：** 思路自然，递归结构清晰。
- **缺点：** 有栈开销，实际性能通常不如自底向上迭代。

---

### 解法四：纯递归（不推荐）

```java
public int climbStairsNaive(int n) {
    if (n <= 2) return n;
    return climbStairsNaive(n - 1) + climbStairsNaive(n - 2);
}
```

- **时间复杂度：** `O(2^n)`（指数级，存在大量重复计算）
- **空间复杂度：** `O(n)`（递归深度）
- **仅用于理解状态转移，不可在 n 较大时使用。**

## 四、解法对比表

| 解法 | 实现方式 | 时间 | 空间 | 优点 | 缺点 | 推荐指数 |
|------|----------|------|------|------|------|----------|
| 滚动变量 DP | 三个变量迭代 | `O(n)` | `O(1)` | 空间最优、速度快 | 需理解状态压缩 | ★★★★★ |
| 数组 DP | `dp[i] = dp[i-1] + dp[i-2]` | `O(n)` | `O(n)` | 最直观 | 空间浪费 | ★★★★☆ |
| 记忆化搜索 | 递归 + memo | `O(n)` | `O(n)` | 结构清晰 | 递归栈开销 | ★★★☆☆ |
| 纯递归 | 直接递归 | `O(2^n)` | `O(n)` | 最易理解 | 严重超时 | ★☆☆☆☆ |

**推荐：** 生产和面试均优先使用「滚动变量 DP」。

## 五、易错点总结

1. **边界处理**：`n=1` 和 `n=2` 必须特殊返回，否则循环从 3 开始时会出错。
2. **变量更新顺序**：`curr = prev2 + prev1` 之后，必须先 `prev2 = prev1`，再 `prev1 = curr`，顺序不能反。
3. **循环范围**：`for (int i = 3; i <= n; i++)`，注意是 `<= n` 而不是 `< n`。
4. **初始值对应关系**：
   - `prev2` 代表 `f(i-2)`（较早的状态）
   - `prev1` 代表 `f(i-1)`（较新的状态）
5. **结果变量**：循环结束后返回 `curr`（或 `prev1`，视实现而定），不要返回 `prev2`。
6. **n=45 的数值范围**：虽然能用 `int`，但如果题目范围扩大到更大 n，就必须改用 `long` 或 `BigInteger`。
7. **不要试图用数学公式直接算**：虽然有通项公式（Binet 公式），但浮点精度问题在 n=45 时已经不可靠，不如迭代稳健。
