# LeetCode 88. 合并两个有序数组（Merge Sorted Array）解题文档

## 一、题目描述

给你两个按**非递减顺序**排列的整数数组 `nums1` 和 `nums2`，另有两个整数 `m` 和 `n`，分别表示 `nums1` 和 `nums2` 中的元素数目。

请你**合并** `nums2` 到 `nums1` 中，使合并后的数组同样按非递减顺序排列。

**注意：** 合并后的数组不应由函数返回，而是**原地存储在 `nums1` 中**。`nums1` 的初始长度为 `m + n`，前 `m` 个元素是有效元素，后 `n` 个为 `0`（占位，应忽略）；`nums2` 的长度为 `n`。

**示例：**

| nums1 | m | nums2 | n | 输出（nums1） | 说明 |
|-------|---|-------|---|---------------|------|
| `[1,2,3,0,0,0]` | 3 | `[2,5,6]` | 3 | `[1,2,2,3,5,6]` | 正常合并 |
| `[1]` | 1 | `[]` | 0 | `[1]` | nums2 为空 |
| `[0]` | 0 | `[1]` | 1 | `[1]` | nums1 无有效元素 |

**链接：** https://leetcode-cn.com/problems/merge-sorted-array

---

## 二、核心观察

1. **结果要原地写回 `nums1`**，而 `nums1` 尾部正好预留了 `n` 个空位。
2. **若从前往后**双指针合并，写入 `nums1` 前部时会**覆盖掉还没比较的 `nums1` 有效元素**，必须额外开数组——O(n) 空间。
3. **从后往前**合并是关键转化：每次把两数组中**较大的元素**放到 `nums1` 的最末尾空位。被写入的位置永远在已比较区域之后，**绝不会覆盖未处理的数据**，从而做到 O(1) 空间原地合并。
4. **收尾**：循环结束后，
   - 若 `nums2` 还有剩余 → 必须复制到 `nums1` 头部；
   - 若 `nums1` 还有剩余 → **无需任何操作**，它们本就在自己正确的位置上。

---

## 三、解法

### 解法一：逆向双指针（最优解）

**思路：**
1. 三个指针：`p1 = m-1` 指向 `nums1` 有效元素末尾，`p2 = n-1` 指向 `nums2` 末尾，`p = m+n-1` 指向 `nums1` 整体末尾（写入位置）。
2. 当 `p1 >= 0 && p2 >= 0`：比较 `nums1[p1]` 与 `nums2[p2]`，把**较大者**写到 `nums1[p]`，对应指针与 `p` 同时左移。
3. 出循环后若 `p2 >= 0`，把 `nums2` 剩余元素依次填入 `nums1` 头部；`nums1` 剩余则不处理。

```java
public void merge(int[] nums1, int m, int[] nums2, int n) {
    int p1 = m - 1;        // nums1 有效元素末尾
    int p2 = n - 1;        // nums2 末尾
    int p = m + n - 1;     // 合并后写入位置（nums1 末尾）

    // 从后往前取较大者放到 p
    while (p1 >= 0 && p2 >= 0) {
        if (nums1[p1] > nums2[p2]) {
            nums1[p--] = nums1[p1--];
        } else {
            nums1[p--] = nums2[p2--];
        }
    }

    // 若 nums2 还有剩余，复制到 nums1 头部
    // （若是 nums1 有剩余则无需处理，它们本就在正确位置上）
    while (p2 >= 0) {
        nums1[p--] = nums2[p2--];
    }
}
```

- **时间复杂度：** O(m + n)，每个元素只被访问/写入一次
- **空间复杂度：** O(1)，仅用三个指针，原地操作
- **优点：** 时间、空间双双最优；不需额外数组；写法简洁
- **缺点：** 逆向思维不如正向直观，需理解「为何从后往前不会覆盖」

---

### 解法二（对比参考）：先拼接后排序

**思路：** 把 `nums2` 的 `n` 个元素直接拷到 `nums1` 后 `n` 个位置，再对整个 `nums1` 调 `Arrays.sort`。

```java
public void merge(int[] nums1, int m, int[] nums2, int n) {
    System.arraycopy(nums2, 0, nums1, m, n);
    Arrays.sort(nums1);
}
```

- **时间复杂度：** O((m+n) log(m+n))，排序主导
- **空间复杂度：** O(log(m+n))，排序递归栈
- **优点：** 代码极短，几乎不会写错
- **缺点：** 完全没利用「两数组已有序」这一前提，时间复杂度更差，属于面试中的下策

## 五、易错点总结

1. **必须从后往前**：正向双指针写回 `nums1` 会覆盖未处理元素。逆向是本题的灵魂。
2. **三个指针初值都是「末尾下标」**：`p1=m-1`、`p2=n-1`、`p=m+n-1`，注意是 `m-1` 不是 `m`。
3. **收尾只需补 `nums2`**：循环结束时若 `nums1` 有剩余可直接忽略（已在原位）；只有 `nums2` 剩余才需要继续拷贝。漏掉「补 nums2」会在 `nums1` 全部更大时出错（如 `nums2=[0]`、`nums1=[1]`、m=0）。
4. **相等时取谁都对**：`nums1[p1] > nums2[p2]` 用严格大于，相等走 else 分支取 `nums2`，结果依然有序，不影响正确性。
5. **空数组边界**：`n=0` 时第一个循环不进入、第二个也不进入，`nums1` 原样即可；`m=0` 时全靠第二个循环把 `nums2` 拷过来。
