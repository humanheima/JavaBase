# LeetCode 28. 找出字符串中第一个匹配项的下标（Find the Index of the First Occurrence in a String）解题文档

## 一、题目描述

给你两个字符串 `haystack` 和 `needle`，请你在 `haystack` 字符串中找出 `needle` 字符串的第一个匹配项的下标（下标从 0 开始）。如果 `needle` 不是 `haystack` 的一部分，则返回 -1。

**示例：**

| haystack | needle | 输出 | 说明 |
|----------|--------|------|------|
| `"sadbutsad"` | `"sad"` | `0` | 第一个 "sad" 出现在下标 0 |
| `"leetcode"` | `"leeto"` | `-1` | 不存在匹配 |
| `"hello"` | `"ll"` | `2` | "ll" 出现在下标 2 |
| `"mississippi"` | `"issip"` | `4` | 中间匹配 |

**链接：** https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/

---

## 二、核心观察

1. **本质是「模式串在文本串中的首次出现位置」**：经典字符串匹配问题，可用朴素法或高效的 KMP、Rabin-Karp、Sunday 等算法解决。
2. **暴力法简单但有重复比较**：当匹配失败时，`i` 只前进 1，`j` 归零，导致大量已匹配前缀被重复扫描（如 `aaaaa` 匹配 `aaa`）。
3. **KMP 的核心思想**：利用模式串自身的「前缀 = 后缀」信息（next 数组 / LPS 数组），匹配失败时**只回退模式串指针 j**，文本串指针 i 绝不回退，实现线性时间。
4. **next 数组的意义**：`next[j]` 表示 `needle[0..j-1]` 的最长真前缀与真后缀的长度。利用它可以在失配时跳过不可能的起始位置。

---

## 三、解法

### 解法一：暴力匹配（Brute Force）

**思路：**

1. 若 `needle` 为空，按惯例返回 0；若 `haystack` 长度小于 `needle`，直接返回 -1。
2. 对 `haystack` 中每一个可能的起始位置 `i`（范围 `0` 到 `n-m`），从该位置开始与 `needle` 逐字符比较。
3. 完全匹配则立即返回 `i`；任一字符不等则提前跳出内层循环，尝试下一个 `i`。
4. 全部位置都失败则返回 -1。

**关键代码：**

```java
public int strStr(String haystack, String needle) {
    if (haystack == null || needle == null) return -1;
    int n = haystack.length();
    int m = needle.length();
    if (m == 0) return 0;
    if (n < m) return -1;

    for (int i = 0; i <= n - m; i++) {
        boolean match = true;
        for (int j = 0; j < m; j++) {
            if (haystack.charAt(i + j) != needle.charAt(j)) {
                match = false;
                break;
            }
        }
        if (match) return i;
    }
    return -1;
}
```

- **时间复杂度：** `O((n - m + 1) · m)` 最坏情况（如 `aaaaa...` 匹配 `aaa...`）。
- **空间复杂度：** `O(1)`。
- **优点：** 代码极简、无需预处理、易于理解和实现。
- **缺点：** 最坏时间复杂度高，在重复字符场景下效率极差。

### 解法二：KMP 算法（推荐）

**思路：**

1. **预处理阶段**：对 `needle`（模式串）构建 `next` 数组。
   - `next[j]` = `needle[0..j-1]` 的最长「真前缀」与「真后缀」相等的长度。
   - 使用双指针 `len` 和 `i` 线性计算。
2. **匹配阶段**：
   - `i` 遍历 `haystack`，`j` 指向 `needle` 当前待匹配位置。
   - 字符相等：`i++`、`j++`，`j == m` 时匹配成功，返回 `i - m`。
   - 字符不等：
     - 若 `j > 0`，令 `j = next[j-1]`（回退到可能的前缀位置）；
     - 若 `j == 0`，只能 `i++`。
3. 整个过程 `i` 只前进不回退，故总比较次数 ≤ `n + m`。

**next 数组构造演示**（`needle = "abab"`）：

```
j=0: next[0]=0（约定）
j=1: 'b' != 'a' → next[1]=0
j=2: 'a' == 'a' → len=1, next[2]=1
j=3: 'b' == 'b' → len=2, next[3]=2
next = [0, 0, 1, 2]
```

**关键代码（匹配 + 构造）：**

```java
public int strStrKMP(String haystack, String needle) {
    // ... 边界处理
    int[] next = buildNext(needle);
    int i = 0, j = 0;
    while (i < haystack.length()) {
        if (haystack.charAt(i) == needle.charAt(j)) {
            i++; j++;
            if (j == needle.length()) return i - needle.length();
        } else {
            if (j != 0) j = next[j - 1];
            else i++;
        }
    }
    return -1;
}

private int[] buildNext(String pat) {
    int m = pat.length();
    int[] next = new int[m];
    int len = 0, i = 1;
    while (i < m) {
        if (pat.charAt(i) == pat.charAt(len)) {
            len++;
            next[i] = len;
            i++;
        } else {
            if (len != 0) len = next[len - 1];
            else { next[i] = 0; i++; }
        }
    }
    return next;
}
```

- **时间复杂度：** `O(n + m)` —— 预处理 `O(m)` + 匹配 `O(n)`。
- **空间复杂度：** `O(m)` —— 需要存储 next 数组。
- **优点：** 线性时间、最坏情况极优、是字符串匹配的标准高效算法之一；`i` 不回退适合流式输入。
- **缺点：** 需要理解 next 数组的含义和构造过程，代码比暴力长；对极短字符串常数因子可能不如暴力。

## 四、解法对比表

| 解法 | 核心思路 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 | 推荐场景 |
|------|----------|------------|------------|------|------|----------|
| 暴力匹配 | 每个位置逐字符比对 | `O((n-m+1)m)` | `O(1)` | 极简、直观 | 重复字符场景极慢 | 字符串极短、快速实现 |
| KMP | 前缀表 + 指针不回退 | `O(n + m)` | `O(m)` | 线性最优、通用 | 需理解 next 数组 | 面试首选、生产环境通用解法 |

**推荐：** 面试中优先写 KMP（或至少能讲清楚思路）；实际工程中若字符串很短可直接用暴力或语言内置 `indexOf`。

## 五、易错点总结

1. **next 数组下标与长度关系**：`next[j]` 对应的是前 `j` 个字符（`0..j-1`）的最长前缀后缀长度。构造时 `i` 从 1 开始，`next[0]` 固定为 0。
2. **失配回退时 `j` 可能多次跳转**：`j = next[j-1]` 可能链式跳转，不能只跳一次就认为安全。
3. **返回位置是起始下标**：匹配成功时是 `i - m`，不是 `i` 或 `j`。
4. **空 needle 的处理**：按题目惯例通常返回 0，代码中应显式处理（本实现已处理）。
5. **needle 长度大于 haystack**：必须提前返回 -1，避免越界。
6. **next 数组构造中的 `len = next[len-1]`**：这是 KMP 的精髓，模拟「模式串自己和自己匹配」的过程，写错会导致整个算法失效。
