# LeetCode 101. 对称二叉树（Symmetric Tree）解题文档

## 一、题目描述

给你一个二叉树的根节点 `root`，检查它是否轴对称（即与它的镜像相同）。

如果一棵二叉树和它的镜像一样，那么它是对称的。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `[1,2,2,3,4,4,3]` | `true` | 经典对称树：左右子树镜像 |
| `[1,2,2,null,3,null,3]` | `false` | 结构对称但对应节点值不等（3 和 3 的位置错位） |
| `[]`（`null`） | `true` | 空树视为对称 |
| `[1]` | `true` | 单节点树对称 |

**链接：** https://leetcode.cn/problems/symmetric-tree/

---

## 二、核心观察

1. **对称的本质是镜像**：左子树的结构和值要与右子树的「镜像」完全一致。
2. 递归分解为成对节点 `(left, right)` 的比较：
   - 都为空 → 对称（递归出口）
   - 恰好一个为空 → 结构不对称
   - 都非空 → 值相等 **且** `left.left` 与 `right.right` 对称 **且** `left.right` 与 `right.left` 对称
3. 遍历顺序上，可以把「正常前序（根-左-右）」与「镜像前序（根-右-左）」的序列进行比较，若完全一致则对称。
4. 迭代同样需要「成对」处理，只是入队子节点时要按镜像位置配对（左左↔右右、左右↔右左）。

---

## 三、两种解法

### 解法一：递归（DFS）

**思路：**
1. 根为 `null` 直接返回 `true`（空树对称）。
2. 调用辅助函数 `isMirror(left, right)` 比较左右子树是否镜像。
3. 在 `isMirror` 中：
   - 两个都为 `null` → 返回 `true`
   - 其中一个为 `null` → 返回 `false`
   - 两个都非空 → 比较 `val` 相等，并且递归比较 `(left.left, right.right)` 和 `(left.right, right.left)`

```java
public static boolean isSymmetric(TreeNode root) {
    if (root == null) return true;
    return isMirror(root.left, root.right);
}

private static boolean isMirror(TreeNode left, TreeNode right) {
    if (left == null && right == null) return true;
    if (left == null || right == null) return false;

    return left.val == right.val
            && isMirror(left.left, right.right)
            && isMirror(left.right, right.left);
}
```

- **时间复杂度：** O(n)，每个节点最多被访问一次
- **空间复杂度：** O(n)，递归栈深度，最坏情况（链状树）为 O(n)，平衡树为 O(log n)
- **优点：** 代码简洁直观，与「镜像」定义完全对应，可读性强
- **缺点：** 依赖系统调用栈，极深的树可能栈溢出

### 解法二：迭代（BFS，队列成对比较）

**思路：**
1. 根为 `null` 返回 `true`。
2. 用一个队列，将根的左右孩子 **成对** 入队：`offer(left)`、`offer(right)`。
3. 循环中每次 **成对** 出队两个节点 `left`、`right`：
   - 都为空 → `continue`
   - 恰好一个为空 → `return false`
   - 值不相等 → `return false`
4. 把子节点按**镜像位置**成对入队：
   - `left.left` 与 `right.right`
   - `left.right` 与 `right.left`
5. 队列处理完仍无差异 → 返回 `true`。

```java
public boolean isSymmetricIterative(TreeNode root) {
    if (root == null) return true;

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root.left);
    queue.offer(root.right);

    while (!queue.isEmpty()) {
        TreeNode left = queue.poll();
        TreeNode right = queue.poll();

        if (left == null && right == null) continue;
        if (left == null || right == null) return false;
        if (left.val != right.val) return false;

        queue.offer(left.left);
        queue.offer(right.right);
        queue.offer(left.right);
        queue.offer(right.left);
    }

    return true;
}
```

- **时间复杂度：** O(n)
- **空间复杂度：** O(n)，队列最坏情况下存储一层的所有节点（宽度优先）
- **优点：** 不依赖系统栈，可处理极深树；显式控制遍历过程
- **缺点：** 必须严格维护「成对入队/出队」和「镜像子节点配对」的不变式，顺序稍错就会比较错位

## 四、两种解法对比

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、递归 DFS | 根同 + 左左↔右右 + 左右↔右左 递归 | O(n) | O(n) 栈 | 最简洁、贴合镜像定义 | 深树有栈溢出风险 |
| 二、迭代 BFS | 队列成对 + 镜像位置入队 | O(n) | O(n) 队列 | 不依赖系统栈 | 需小心镜像配对顺序 |

---

## 五、易错点总结

1. **空节点处理顺序**：必须先判断 `left == null && right == null`，再判断 `left == null || right == null`，最后才访问 `val`。顺序错误会导致 NPE。
2. **镜像配对方向极易写反**：
   - 递归：`isMirror(left.left, right.right)` 和 `isMirror(left.right, right.left)`
   - 迭代：`offer(left.left); offer(right.right);` `offer(left.right); offer(right.left);`
   写成普通「相同树」的配对（left.left ↔ right.left）就会全部判错。
3. **迭代必须把 `null` 孩子也入队**：只有把 `null` 也成对 offer，才能在出队时通过「一空一非空」检测出结构不对称。
4. **根节点单独处理**：根为 `null` 直接返回 `true`，之后只比较它的左右子树。
5. **测试覆盖**：至少要覆盖「完全对称」「结构不对称」「值不对称」「全空」「单节点」「所有值相同但结构不对称」这几类。
