# LeetCode 9. 回文数（Palindrome Number）解题文档

## 一、题目描述

给你一个整数 `x`，如果 `x` 是一个回文整数，返回 `true`；否则返回 `false`。

回文数是指正序（从左向右）和倒序（从右向左）读都一样的整数。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `121` | `true` | 正读倒读都是 121 |
| `-121` | `false` | 倒读为 `121-`，因含负号不是回文 |
| `10` | `false` | 倒读为 `01`，不是回文 |

**链接：** https://leetcode-cn.com/problems/palindrome-number

---

## 二、核心观察

1. **负数一定不是回文数**：因为负号 `-` 在最前面，倒读时跑到末尾，永远不对称。
2. **末位为 0 且本身非 0 的数一定不是回文数**：如 `10`、`120`，因为首位不可能是 0。
3. 判断回文本质是：**数字在镜像（反转）后是否与原值相同**。

---

## 三、三种解法

### 解法一：字符串 + 双指针

**思路：**
1. 负数直接返回 `false`。
2. 把整数转成字符串。
3. 用左右两个指针从两端向中间夹逼，逐位比较，遇到不相等立即返回 `false`。

```java
public static boolean isPalindrome(int x) {
    if (x < 0) return false;
    String s = String.valueOf(x);
    int left = 0, right = s.length() - 1;
    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) return false;
        left++;
        right--;
    }
    return true;
}
```

- **时间复杂度：** O(n)，n 为位数
- **空间复杂度：** O(n)，需要额外的字符串空间
- **优点：** 直观易懂
- **缺点：** 额外占用字符串空间

---

### 解法二：只反转后半部分（官方推荐）

**思路：**
1. 处理特殊情况：负数、末位为 0 的非 0 数返回 `false`；0 返回 `true`。
2. 只反转数字的**后半部分**，当「反转部分 `reversed` ≥ 剩余部分 `x`」时停止，说明已处理到中间。
3. 比较前后两半：
   - 偶数位：`x == reversed`
   - 奇数位：`x == reversed / 10`（去掉中间位）

```java
public static boolean isPalindrome2(int x) {
    if (x < 0) return false;
    if (x == 0) return true;
    if (x % 10 == 0) return false;   // 末位为 0 的非 0 数

    int reversed = 0;
    while (x > reversed) {
        reversed = reversed * 10 + x % 10;
        x = x / 10;
    }
    // 偶数位 x == reversed；奇数位 x == reversed / 10
    return x == reversed || x == reversed / 10;
}
```

- **时间复杂度：** O(log₁₀x)，只处理一半位数
- **空间复杂度：** O(1)
- **优点：** 不会溢出、只算一半，效率最高
- **缺点：** 边界特判较多，奇偶位逻辑需要理解

---

### 解法三：完整反转整个数字（本次新增）

**思路：**
1. 负数直接返回 `false`。
2. 把**整个数字**反转得到 `reversed`，例如 `123 → 321`、`121 → 121`。
3. 反转可能超出 `int` 范围（如 `2147483647` 反转为 `7463847412`），所以用 `long` 存储 `reversed`，规避溢出。
4. 反转后与原数比较：相等即回文。

```java
public static boolean isPalindrome(int x) {
    if (x < 0) return false;

    long reversed = 0;
    int origin = x;
    while (x != 0) {
        reversed = reversed * 10 + x % 10; // 取末位拼到 reversed
        x /= 10;                           // 去掉末位
    }
    return reversed == origin;
}
```

- **时间复杂度：** O(log₁₀x)
- **空间复杂度：** O(1)
- **优点：** 写法最简洁，无需特判末位 0
- **缺点：** 需借助 `long` 处理溢出

---

## 四、三种解法对比

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、字符串双指针 | 转字符串后两端夹逼 | O(n) | O(n) | 直观 | 额外空间 |
| 二、反转后半部分 | 只算一半再比较 | O(log x) | O(1) | 不溢出、效率高 | 边界多 |
| 三、完整反转 | 反转全部 + long | O(log x) | O(1) | 最简洁 | 依赖 long |

---

## 五、易错点总结

1. **负数**：必须最先排除。
2. **末位为 0**：除 0 本身外（如 `10`、`100`）都不是回文。
3. **整数溢出**：完整反转法必须用 `long`，否则 `int` 溢出会导致误判。
4. **奇偶位**：只反转一半时，奇数位要 `reversed / 10` 去掉中间位再比较。
