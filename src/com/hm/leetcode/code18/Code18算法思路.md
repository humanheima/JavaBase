# LeetCode 18. 四数之和（4Sum）解题文档

## 一、题目描述

给你一个由 `n` 个整数组成的数组 `nums` 和一个目标值 `target`。请你找出并返回满足下述条件且**不重复**的四元组 `[nums[a], nums[b], nums[c], nums[d]]`：

- `0 <= a, b, c, d < n`，且 `a`、`b`、`c`、`d` 互不相同；
- `nums[a] + nums[b] + nums[c] + nums[d] == target`。

可以按任意顺序返回答案。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `nums = [1,0,-1,0,-2,2], target = 0` | `[[-2,-1,1,2],[-2,0,0,2],[-1,0,0,1]]` | 三个不重复四元组 |
| `nums = [2,2,2,2,2], target = 8` | `[[2,2,2,2]]` | 重复元素只算一组 |

**提示：**
- `1 <= nums.length <= 200`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`

**链接：** https://leetcode.cn/problems/4sum/

---

## 二、核心观察 / 关键点

1. **是「三数之和」的再升一维**：15 题是「固定 1 个数 + 双指针」，本题是「固定 2 个数 + 双指针」。骨架完全一样，只是外面多套一层循环。
2. **排序是去重与双指针的前提**：排序后相等元素相邻，去重只需「跳过与前一个相同的值」。
3. **必须用 `long` 防溢出**：`nums[i]` 与 `target` 可达 `±10^9`，四个数相加最大约 `4×10^9`，**超过 `int` 上限 `2.147×10^9`**。求和时要 `(long)` 强转，否则会溢出导致误判。
4. **四层去重缺一不可**：`nums[i]`（第一层）、`nums[j]`（第二层）、命中后 `nums[left]` / `nums[right]`（双指针层）都要跳过重复，任意一处漏掉都会产生重复四元组。
5. **可选剪枝**：固定 `i`（或 `i,j`）后，用「当前最小四数和」「当前最大四数和」与 `target` 比较，可提前 `break` / `continue`，大幅减少无效遍历。

---

## 三、解法

### 解法一：排序 + 双指针 + 剪枝（推荐，源码 `fourSum`）

**思路：**
1. 排序。外层固定 `nums[i]`（`i: 0 → n-4`），内层固定 `nums[j]`（`j: i+1 → n-3`）。
2. 每层都先跳过重复固定值，再做两条剪枝：
   - **下界剪枝**：若「`i`（或 `i,j`）配上紧邻的最小几个数」之和已 `> target`，后面只会更大 → `break`。
   - **上界剪枝**：若「`i`（或 `i,j`）配上末尾最大几个数」之和仍 `< target`，本轮不可能命中 → `continue`。
3. 在 `[j+1, n-1]` 上用左右指针找 `sum == target`；命中后跳过两侧重复值再各走一步。

**关键代码片段：**（与源文件 `Code18.java#fourSum` 一致）

```java
Arrays.sort(nums);
int n = nums.length;
for (int i = 0; i < n - 3; i++) {
    if (i > 0 && nums[i] == nums[i - 1]) continue;                       // 去重 i
    if ((long) nums[i] + nums[i+1] + nums[i+2] + nums[i+3] > target) break;       // 下界剪枝
    if ((long) nums[i] + nums[n-3] + nums[n-2] + nums[n-1] < target) continue;    // 上界剪枝

    for (int j = i + 1; j < n - 2; j++) {
        if (j > i + 1 && nums[j] == nums[j - 1]) continue;              // 去重 j
        if ((long) nums[i] + nums[j] + nums[j+1] + nums[j+2] > target) break;     // 下界剪枝
        if ((long) nums[i] + nums[j] + nums[n-2] + nums[n-1] < target) continue;  // 上界剪枝

        int left = j + 1, right = n - 1;
        while (left < right) {
            long sum = (long) nums[i] + nums[j] + nums[left] + nums[right]; // long 防溢出
            if (sum == target) {
                result.add(Arrays.asList(nums[i], nums[j], nums[left], nums[right]));
                while (left < right && nums[left] == nums[left + 1]) left++;   // 去重 left
                while (left < right && nums[right] == nums[right - 1]) right--; // 去重 right
                left++;
                right--;
            } else if (sum < target) {
                left++;
            } else {
                right--;
            }
        }
    }
}
```

- **时间复杂度：** O(n³)。两层循环 O(n²) × 双指针 O(n)；排序 O(n log n) 被吞没。
- **空间复杂度：** O(1)（不含排序栈与结果集）。
- **优点：** 剪枝后实测很快，是本题标准最优解。
- **缺点：** 剪枝边界表达式较多，写错下标容易越界。

### 解法二：排序 + 双指针（基础版，无剪枝，源码 `fourSum2`）

**思路：** 与解法一完全相同，只是去掉了上/下界剪枝。逻辑更短、更易读，正确性一致，只是少了提前退出的加速。

### 解法三：暴力四重循环 + 去重（仅作对照基准）

**思路：** 排序后四层循环枚举所有 `a<b<c<d`，和为 `target` 时用 `Set` 去重。

- **时间复杂度：** O(n⁴)。
- **空间复杂度：** O(去重集合)。
- **优点：** 直白，作正确性基准交叉验证双指针。
- **缺点：** n=200 时约 6.4×10⁷ 组合，偏慢，非首选。

## 五、易错点总结

1. **整数溢出（最重要）**：四数和最大约 `4×10^9` 超过 `int`，求和与剪枝判断都要 `(long)` 强转。不转的话 `1e9 + 1e9 + 1e9 + 1e9` 会溢出成 `-294967296`，可能误命中负的 `target`（测试用例已专门覆盖此场景）。
2. **四处去重**：`i`、`j`、命中后的 `left` / `right` 都要跳过重复值；尤其 `j` 的去重条件是 `j > i + 1`（不是 `j > 0`），否则会错杀 `j == i+1` 这个合法起点。
3. **剪枝要先判去重再判边界**：顺序不当可能在 `break` 前漏掉合法分支。下界用 `break`（后面只会更大），上界用 `continue`（换下一个固定数）。
4. **下标边界**：外层到 `n-3`，内层到 `n-2`，剪枝里用到 `i+1,i+2,i+3` 与 `n-1,n-2,n-3`，要确保 `n >= 4` 时不越界（开头已 `nums.length < 4` 直接返回）。
5. **去重非用 Set 不可吗**：双指针法靠「排序 + 跳过相邻相同」即可去重，无需额外 `Set`；只有暴力法才借助 `Set`。

## 六、专题：双指针到底「固定谁、动谁」（3Sum vs 4Sum）

一个常见误解是把它理解成「三数固定头尾向中间找，四数固定头部两个向后找」。准确说法是：**两者的双指针都是左右向中间夹逼，区别只在于「外面固定了几个数」。**

### 6.1 三数之和：固定 1 个 + 双指针夹逼

```
[ i ][ left → ......... ← right ]
 固定    这两个指针相向移动
```

固定的只有 `nums[i]`（三元组里最小的那个）；`left = i+1`、`right = n-1` 是**会移动的双指针**，相向夹逼。所以「头尾」不是固定的，恰恰是那两个动的指针。

### 6.2 四数之和：固定 2 个 + 双指针夹逼

```
[ i ][ j ][ left → ..... ← right ]
 固定  固定   这两个指针相向移动
```

固定 `nums[i]`、`nums[j]`；`left = j+1`、`right = n-1` 依然是**相向夹逼**，不是「向后找」。只是起点 `left` 挪到了 `j` 后面。

### 6.3 统一规律（kSum）

| 问题 | 固定几个数 | 最后两个数 | 总复杂度 |
|------|-----------|-----------|---------|
| 两数之和（有序） | 固定 0 个 | left/right 夹逼 | O(n) |
| 三数之和 | 固定 1 个 | left/right 夹逼 | O(n²) |
| 四数之和 | 固定 2 个 | left/right 夹逼 | O(n³) |
| k 数之和 | 固定 k−2 个 | left/right 夹逼 | O(n^(k−1)) |

**核心一句话**：不管几数之和，**最后两个数永远是「左右指针相向夹逼」**；增加一个数，就在外面多套一层 `for` 去固定它，复杂度随之乘一个 `n`。

之所以双指针能 O(n) 收尾，靠的是**数组已排序**：`sum` 偏小就 `left++`（变大），偏大就 `right--`（变小）。这个性质与固定了几个数无关，所以 3Sum、4Sum 的内层双指针逻辑一模一样。
