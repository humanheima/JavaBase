# LeetCode 5. 最长回文子串

> https://leetcode.cn/problems/longest-palindromic-substring/

## 一、问题描述

给定一个字符串 `s`，找到 `s` 中最长的**回文子串**。

回文：正着读和倒着读完全一样的字符串，如 `"aba"`、`"bb"`。注意是**子串**（连续），不是子序列。

| 输入 | 输出 | 解释 |
| --- | --- | --- |
| `"babad"` | `"bab"` | `"aba"` 也是有效答案 |
| `"cbbd"` | `"bb"` | |
| `"a"` | `"a"` | 单字符本身就是回文 |
| `""` | `""` | 空串 |

## 二、核心思想：中心扩展法

暴力解法是枚举所有子串 `O(n²)` 个，再逐个判断是否回文 `O(n)`，总共 `O(n³)`，太慢。

换个角度：**每个回文串都有一个对称中心，从中心向两边扩展时，字符是镜像相等的**。

> 与其枚举"两个端点"，不如枚举"中心"，再向两边扩展，能扩多远就有多长的回文。

这样把判断回文的过程和定位回文的过程合并了，复杂度降到 `O(n²)`。

### 关键：中心有两种

回文长度有奇偶之分，对称中心也分两类：

- **奇数长度**（如 `"aba"`）：中心是**一个字符**，从 `(i, i)` 开始扩展。
- **偶数长度**（如 `"bb"`）：中心是**两个字符之间的缝隙**，从 `(i, i+1)` 开始扩展。

所以对每个位置 `i`，都要尝试这两种中心，取较长的那个。

长度为 `n` 的字符串，一共有 `2n - 1` 个可能的中心（`n` 个单字符中心 + `n-1` 个缝隙中心）。

## 三、向两边扩展

扩展函数 `expandAroundCenter(s, left, right)`：

```
只要 left、right 没越界，且 s[left] == s[right]：
    left--   （向左扩）
    right++  （向右扩）
循环结束后，[left+1, right-1] 就是以该中心能得到的最长回文。
```

循环结束时 `left` 和 `right` 已经各自多走了一步（指向不相等或越界的位置），
所以**回文长度 = right - left - 1**。

> 为什么是 `right - left - 1` 而不是 `right - left + 1`？
> 因为退出循环时，真正的回文区间是 `[left+1, right-1]`，
> 长度 = `(right-1) - (left+1) + 1` = `right - left - 1`。

## 四、由长度反推起始位置

主循环里拿到本次回文长度 `len` 和中心下标 `i` 后，需要算出回文的起始下标 `start`：

```java
start = i - (len - 1) / 2;
```

中心在 `i`，回文向左扩了 `(len-1)/2` 步，所以起点就是 `i` 减去左侧步数。

以 `"babad"` 的 `"bab"` 为例：中心 `i = 1`，`len = 3`，`start = 1 - (3-1)/2 = 0`，
即从下标 0 开始、长度 3，正是 `"bab"`。

> 这个公式对奇偶都成立。偶数中心 `(i, i+1)` 时，扩展函数返回的 `len` 是偶数，
> `(len-1)/2` 的整数除法刚好补偿了中心偏右半格的差异。

## 五、执行流程示例（"babad"）

| i | 奇中心 (i,i) 长度 | 偶中心 (i,i+1) 长度 | 本轮最长 | maxLength | start |
| --- | --- | --- | --- | --- | --- |
| 0 | `b` → 1 | `ba` → 0 | 1 | 1 | 0 |
| 1 | `bab` → 3 | `ab` → 0 | 3 | 3 | 0 |
| 2 | `aba` → 3 | `ba` → 0 | 3 | 3（不更新） | 0 |
| 3 | `a` → 1 | `ad` → 0 | 1 | 3 | 0 |
| 4 | `d` → 1 | 越界 → 0 | 1 | 3 | 0 |

最终 `s.substring(0, 0 + 3)` = `"bab"`。

> `i = 2` 时也找到了长度为 3 的 `"aba"`，但因为 `len > maxLength` 不成立（相等不更新），
> 保留了先找到的 `"bab"`，这也是为什么本题允许返回多个有效答案之一。

## 六、代码

```java
public String longestPalindrome(String s) {
    if (s == null || s.isEmpty()) return "";

    int start = 0;      // 最长回文串的起始位置
    int maxLength = 0;  // 最长回文串的长度

    for (int i = 0; i < s.length(); i++) {
        int len1 = expandAroundCenter(s, i, i);      // 奇数长度回文
        int len2 = expandAroundCenter(s, i, i + 1);  // 偶数长度回文
        int len = Math.max(len1, len2);

        if (len > maxLength) {
            maxLength = len;
            start = i - (len - 1) / 2;  // 由中心和长度反推起点
        }
    }

    return s.substring(start, start + maxLength);
}

// 以 left、right 为中心向两边扩展，返回最长回文长度
private int expandAroundCenter(String s, int left, int right) {
    while (left >= 0 && right < s.length() && s.charAt(left) == s.charAt(right)) {
        left--;
        right++;
    }
    return right - left - 1;
}
```

## 七、复杂度分析

- **时间复杂度** `O(n²)`：`2n - 1` 个中心，每个中心最多扩展 `O(n)`。
- **空间复杂度** `O(1)`：只用了几个下标变量，没有额外数据结构。

## 八、其他解法对比

| 解法 | 时间 | 空间 | 特点 |
| --- | --- | --- | --- |
| 暴力枚举 | `O(n³)` | `O(1)` | 枚举所有子串再判断回文，最直观但最慢 |
| 动态规划 | `O(n²)` | `O(n²)` | `dp[i][j]` 表示 `s[i..j]` 是否回文，由短区间推长区间 |
| **中心扩展（本文）** | `O(n²)` | `O(1)` | 时间同 DP 但省空间，思路也更直观 |
| Manacher | `O(n)` | `O(n)` | 最优解，用对称性复用已算结果，但实现较复杂 |

## 九、小结

中心扩展法的套路：

1. 回文一定有对称中心，**枚举中心**而非端点；
2. 中心分**单字符（奇）** 和 **缝隙（偶）** 两种，每个位置都试一遍；
3. 从中心向两边扩展，`s[left] == s[right]` 就继续，否则停；
4. 用 `right - left - 1` 得到长度，用 `i - (len-1)/2` 反推起点，更新最长答案。

记住两个易错点：扩展退出后**长度是 `right - left - 1`**，以及**别忘了偶数中心 `(i, i+1)`**。
