# LeetCode 24. 两两交换链表中的节点（Swap Nodes in Pairs）解题文档

## 一、题目描述

给你一个链表，两两交换其中相邻的结点，并返回交换后链表的头结点。

你必须在 **不修改结点内部的值** 的情况下完成本题（即只能进行结点交换，而不是交换 `val`）。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `[1,2,3,4]` | `[2,1,4,3]` | (1,2) 交换、(3,4) 交换 |
| `[]` | `[]` | 空链表 |
| `[1]` | `[1]` | 只有一个结点，不交换 |
| `[1,2,3]` | `[2,1,3]` | 末尾落单的 3 保持不动 |

**链接：** https://leetcode.cn/problems/swap-nodes-in-pairs/

---

## 二、核心观察

1. **以"对"为单位处理**：链表从头开始，每两个结点为一组做交换；不足两个（剩 0 或 1 个）的尾巴保持原样。
2. **交换的是指针，不是值**：题目明确禁止改 `val`，必须真正改变结点间的 `next` 指向。
3. **每组交换涉及三根指针的改写**，且要记住"上一组的尾结点"以便把它接到本组新的头上 —— 这正是 **哑结点（dummy）** 的用武之地，它让"头结点也有前驱"，避免对第一组做特判。
4. **交换一对的本质**：对于 `prev -> A -> B -> rest`，交换后变成 `prev -> B -> A -> rest`，需要按正确顺序改三处指针，否则会丢链或成环。

---

## 三、解法

### 解法一：迭代 + 哑结点（推荐）

**思路：**

1. 建哑结点 `dummy` 指向 `head`，`prev` 指向 `dummy`，作为每组交换前的"前驱"。
2. 取当前组的两个结点 `first`、`second`，按三步改指针完成交换：
   - `first.next = second.next`（first 指向下一组的头，跨过 second）；
   - `prev.next = second`（前驱接到 second，second 成为本组新头）；
   - `second.next = first`（second 接回 first）。
3. 交换后，本组的尾结点是 `first`，把 `prev` 前移到 `first`，再取下一组的 `first`、`second`。
4. 当 `first` 或 `second` 为空（不足一对）时停止。

**指针变化图示**（处理 `1->2->3->4`，首组）：

```
初始：  dummy -> 1 -> 2 -> 3 -> 4
        prev    first second

① first.next = second.next →  1 -> 3
② prev.next  = second       →  dummy -> 2
③ second.next = first       →  2 -> 1

结果：  dummy -> 2 -> 1 -> 3 -> 4
                     prev'(移到1) 继续下一组
```

**关键代码：**

```java
public ListNode swapPairs(ListNode head) {
    if (head == null || head.next == null) return head;

    ListNode dummy = new ListNode(0);
    ListNode prev = dummy;
    prev.next = head;

    ListNode first = head;
    ListNode second = head.next;
    while (first != null && second != null) {
        first.next = second.next;   // ① first 跨过 second
        prev.next = second;         // ② 前驱接到 second
        second.next = first;        // ③ second 接回 first

        prev = prev.next.next;      // prev 前移到本组尾(原 first)
        first = prev.next;          // 下一组的 first
        if (first != null) {
            second = first.next;    // 下一组的 second
        }
    }
    return dummy.next;
}
```

- **时间复杂度：** `O(n)` —— 每个结点只被处理一次。
- **空间复杂度：** `O(1)` —— 只用了常数个指针。
- **优点：** 原地交换、空间最省，适合大链表。
- **缺点：** 指针改写顺序敏感，容易写错。

---

### 解法二：递归

**思路：**

1. 递归终止：不足两个结点（`head == null` 或 `head.next == null`）直接返回 `head`。
2. 取前两个结点 `first`、`second`，把 **剩余部分递归交换** 的结果接到 `first.next`。
3. 再让 `second.next = first`，返回 `second` 作为这两个结点交换后的新头。

**关键代码：**

```java
public ListNode swapPairsRecursive(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode first = head;
    ListNode second = head.next;
    first.next = swapPairsRecursive(second.next); // 剩余部分递归处理
    second.next = first;                          // second 接回 first
    return second;                                // second 是新头
}
```

- **时间复杂度：** `O(n)`。
- **空间复杂度：** `O(n)` —— 递归栈深度为 `n/2`。
- **优点：** 代码极简洁，逻辑清晰自然。
- **缺点：** 链表很长时递归栈可能过深（本题约束 `0 ~ 100` 个结点，无忧）。

---

## 四、解法对比表

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、迭代 + 哑结点 | 三步改指针，逐对交换 | O(n) | O(1) | 省空间、适合大链表 | 指针顺序易错 |
| 二、递归 | 交换头两个 + 递归剩余 | O(n) | O(n) | 代码最简洁 | 递归栈开销 |

**推荐：** 面试中两种都可，递归更易写对、易讲清；对空间敏感或超长链表用迭代。

---

## 五、易错点总结

1. **必须交换结点而非交换值**：题目禁止改 `val`，只交换 `val` 虽能过测试但不符合题意。
2. **哑结点不可少**：它统一了"第一组"和"后续组"的处理，省去对头结点的特判。
3. **三步改指针的顺序**：`first.next = second.next` 要先做（先记住下一组入口），否则改了 `prev.next` 或 `second.next` 后会丢失后继。
4. **奇数个结点**：末尾落单的那个结点保持原位，循环条件 `first != null && second != null` 天然处理了这一点。
5. **prev 的前移位置**：交换后本组尾是原来的 `first`，`prev` 要移到这里，不能移错。
6. **空链表 / 单结点**：开头要判 `head == null || head.next == null` 直接返回。
