# LeetCode 8. 字符串转换整数 (atoi)

> https://leetcode.cn/problems/string-to-integer-atoi/

## 一、问题描述

实现一个 `myAtoi(String s)` 函数，把字符串转换成一个 **32 位有符号整数**（类似 C/C++ 的 `atoi`）。

转换规则按顺序执行：

1. **跳过前导空格**（本题空白字符只有空格 `' '`）；
2. **读取符号**：下一个字符若是 `'+'` 或 `'-'`，记录正负；都没有则默认为正；
3. **读取数字**：从当前位置往后读连续的数字字符，直到遇到非数字或字符串结尾，**其余部分一律忽略**；
4. **拼成整数**：把读到的数字转成整数（`"0032" → 32`）；没读到任何数字则为 `0`；
5. **范围截断**：若超出 `[-2^31, 2^31 - 1]`，则钳制到边界——小于 `-2^31` 返回 `-2147483648`，大于 `2^31 - 1` 返回 `2147483647`。

| 输入 | 输出 | 说明 |
| --- | --- | --- |
| `"42"` | `42` | 普通数字 |
| `"   -42"` | `-42` | 跳过前导空格，读到负号 |
| `"4193 with words"` | `4193` | 遇到空格（非数字）停止 |
| `"words and 987"` | `0` | 首个非空字符不是数字/符号，无法转换 |
| `"-91283472332"` | `-2147483648` | 超出下界，钳制到 `-2^31` |
| `"+-12"` | `0` | 第二个符号 `-` 已是非数字，停止，无数字 → 0 |

## 二、核心思想：分阶段顺序扫描

用一个下标 `i` 从左到右一次性扫过字符串，**严格按规则的四个阶段推进**，每个阶段做完 `i` 停在下一阶段的起点：

```
[ 前导空格 ] [ 符号 ] [ 数字数字数字… ] [ 其余忽略 ]
     ↑          ↑           ↑                ↑
   跳过      记录 sign    累加 result      不再处理
```

阶段之间是**线性、不回头**的，所以时间复杂度 `O(n)`。

## 三、四个阶段拆解

### 1. 跳过前导空格

```java
while (i < n && s.charAt(i) == ' ') {
    i++;
}
```

只跳空格 `' '`。注意题目明确：**除前导空格外不要忽略其它任何字符**——所以制表符、数字之间的空格都不跳。

### 2. 读取符号

```java
int sign = 1;
if (i < n && (s.charAt(i) == '+' || s.charAt(i) == '-')) {
    sign = (s.charAt(i) == '-') ? -1 : 1;
    i++;
}
```

符号**最多读一个**。这正是 `"+-12"` 返回 `0` 的原因：读完 `'+'` 后 `i` 指向 `'-'`，而 `'-'` 不是数字，阶段 3 一个数字都读不到，结果为 `0`。

### 3 & 4. 读数字 + 累加 + 判溢出

```java
int result = 0;
while (i < n && s.charAt(i) >= '0' && s.charAt(i) <= '9') {
    int digit = s.charAt(i) - '0';

    // 推入前判断溢出
    if (result > Integer.MAX_VALUE / 10
            || (result == Integer.MAX_VALUE / 10 && digit > 7)) {
        return sign == 1 ? Integer.MAX_VALUE : Integer.MIN_VALUE;
    }

    result = result * 10 + digit;
    i++;
}
return sign * result;
```

字符转数字用 `s.charAt(i) - '0'`（字符 `'0'~'9'` 的 ASCII 连续）。
前导 0 不用特判：`"0032"` 累加得 `0→0→3→32`，自然丢弃前导 0。

## 四、难点：溢出截断

与「整数反转」一样，**在乘 10 之前**判断 `result * 10 + digit` 是否越界，不必借助 `long`。

关键常量 `Integer.MAX_VALUE = 2147483647`（末位 7）：

- 若 `result > MAX / 10`（即 `> 214748364`），乘 10 必然越界；
- 若 `result == MAX / 10`，则 `digit > 7` 时越界。

**为什么只用 MAX 一套判断就够？**
因为我们在累加阶段把数字部分当成**非负**的 `result` 来攒，符号 `sign` 单独保存，最后才相乘。所以：

- 正数越界 → 返回 `Integer.MAX_VALUE`；
- 负数越界 → 返回 `Integer.MIN_VALUE`。

用 `sign == 1 ? MAX : MIN` 一行区分即可。

### 为什么不需要单独判断"小于 MIN_VALUE"？

这是 atoi 和「整数反转」(code7) 写法最大的不同。原因有两层：

**第一层：`result` 在 atoi 里永远是非负的。**

code7 的 `reverse` 把符号**带在 `result` 里**累加，`result` 会真的变成负数，所以必须正负两套判断
（既判 `> MAX/10`，又判 `< MIN/10`）。

而 atoi 把符号**单独存进 `sign`**，累加阶段只攒数字的**绝对值**：

```java
result = result * 10 + digit;   // result 始终 ≥ 0，只往正方向增长
```

`result` 不会变成负数，**没有"负方向"可越界**，自然不需要 `< MIN_VALUE` 的判断——只需判断这个正的 magnitude 会不会超界。

**第二层：用 MAX 这一套阈值卡负数，结果恰好正确。**

负数能取到的最大绝对值是 `2147483648`（`|MIN_VALUE|`），比正数上限 `2147483647` 大 1。
用 `digit > 7`（MAX 阈值）去卡负数，会在边界 `result == MAX/10` 且 `digit == 8` 这一刻"误判"——
但返回值恰好正确：

| 情形 | 实际值 | 是否合法 | `digit>7` 是否触发 | 返回 | 对不对 |
| --- | --- | --- | --- | --- | --- |
| 正数 `2147483648` | 超 MAX | 越界 | 触发 | `MAX_VALUE` = 2147483647 | ✅ |
| 负数 `-2147483648` | = MIN | **合法** | 触发（误判越界） | `MIN_VALUE` = -2147483648 | ✅ 巧合相等 |

负数那行代码**以为**越界提前返回，但钳制返回的 `MIN_VALUE` 恰好等于 `-2147483648` 这个本该算出的合法值，
所以返回依然正确。绝对值 `> 2147483648` 的情况（如 `-91283472332`）正负都是真越界，按 `sign` 返回边界，也对。

> 一句话：符号分离让 `result` 只增不减、没有负方向，故无需 `< MIN` 判断；
> 唯一被 MAX 阈值"误判"的负数边界值 `-2147483648`，又正好等于钳制返回的 `MIN_VALUE`，误判结果反而是对的。
> 代价是少一套分支、少一个魔数 `8`，逻辑更简洁。

## 五、执行流程示例（`"   -42"`）

| 阶段 | i 起点字符 | 动作 | 结束后 i 指向 | 状态 |
| --- | --- | --- | --- | --- |
| 1 跳空格 | `' '` | 连跳 3 个空格 | `'-'` | - |
| 2 读符号 | `'-'` | `sign = -1`，i++ | `'4'` | sign=-1 |
| 3 读数字 | `'4'` | `result = 0*10+4 = 4` | `'2'` | result=4 |
| 3 读数字 | `'2'` | `result = 4*10+2 = 42` | 末尾 | result=42 |

返回 `sign * result = -1 * 42 = -42`。

## 六、复杂度分析

- **时间复杂度** `O(n)`：每个字符至多被访问一次，`n = s.length()`。
- **空间复杂度** `O(1)`：只用了 `i`、`sign`、`result` 几个变量。

## 七、易错点清单

| 易错点 | 说明 |
| --- | --- |
| 只跳前导空格 | 数字中间或后面的空格不跳，遇到即停止（`"4193 with words" → 4193`） |
| 符号只读一个 | `"+-12"`、`"++1"` 第二个符号即终止 → `0` |
| 无有效数字返回 0 | `"words and 987"`、`"   "`、`"+"` 都返回 `0` |
| 前导 0 | `"   +0 123" → 0`（读到 `0` 后遇空格停止，后面的 123 被忽略） |
| 溢出钳制而非返回 0 | 与整数反转不同！atoi 越界要返回 `MAX/MIN`，不是 `0` |
| 事前判溢出 | 乘 10 前判断，避免脏值 |

## 八、小结

字符串转整数（atoi）的套路：

1. **分四阶段线性扫描**：空格 → 符号 → 数字 → 忽略其余，一个下标走到底；
2. 符号**最多一个**，数字**遇非数字即停**；
3. 累加时 `result * 10 + (c - '0')`，前导 0 自然处理；
4. **事前判溢出**，越界按符号**钳制**到 `MAX_VALUE / MIN_VALUE`（注意是钳制，不是返回 0）。
