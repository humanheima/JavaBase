# LeetCode 58. 最后一个单词的长度（Length of Last Word）解题文档

## 一、题目描述

给你一个字符串 `s`，由若干单词组成，单词前后用一些空格字符隔开。返回字符串中**最后一个单词的长度**。

**单词**是指仅由字母组成、不包含任何空格字符的最大子字符串。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `"Hello World"` | `5` | 最后一个单词是 `"World"` |
| `"   fly me   to   the moon  "` | `4` | 最后一个单词是 `"moon"` |
| `"luffy is still joyboy"` | `6` | 最后一个单词是 `"joyboy"` |

**提示：**
- `1 <= s.length <= 10^4`
- `s` 仅由英文字母和空格 `' '` 组成
- `s` 中至少存在一个单词

**链接：** https://leetcode.cn/problems/length-of-last-word/

---

## 二、核心观察

1. **我们只关心最后一个单词**：无需处理前面的所有单词，**从后往前**扫描是最自然的思路。
2. **空格是干扰项**：字符串可能有前导空格、结尾空格、以及单词之间多个连续空格。必须正确跳过它们。
3. **最后一个单词的结束位置就是字符串末尾（或最后一个非空格字符）**。
4. **不需要保存单词本身**：题目只要求长度，因此可以用一个计数器边走边数，无需截取子串。
5. **split 方案的本质**：把问题转成「分割后取最后一个片段的长度」，实现简单但会产生中间数组和字符串对象。

---

## 三、解法

### 解法一：从后往前遍历（推荐，O(1) 空间）

**思路：**

1. 从字符串最后一个字符开始。
2. 第一步：向左跳过所有结尾的空格（处理 trailing spaces）。
3. 第二步：继续向左统计连续的非空格字符，直到遇到空格或到达开头。
4. 统计到的字符个数就是最后一个单词的长度。

这个过程只需要两个阶段的线性扫描，指针只向左移动，不需要任何额外空间。

**关键代码：**

```java
public int lengthOfLastWord(String s) {
    if (s == null || s.isEmpty()) {
        return 0;
    }

    int i = s.length() - 1;

    // 1. 跳过末尾的所有空格
    while (i >= 0 && s.charAt(i) == ' ') {
        i--;
    }

    // 2. 统计最后一个单词的长度
    int len = 0;
    while (i >= 0 && s.charAt(i) != ' ') {
        len++;
        i--;
    }

    return len;
}
```

- **时间复杂度：** `O(n)`，每个字符最多被访问一次。
- **空间复杂度：** `O(1)`，只使用常数个变量。
- **优点：** 空间最优、无中间对象、代码简洁、适合流式或超长字符串。
- **缺点：** 需要仔细处理边界（全空格、单个单词、无空格等）。

### 解法二：trim + split（原实现风格）

**思路：**

1. 调用 `trim()` 去除字符串首尾空格。
2. 使用正则 `split("\\s+")` 按一个或多个空白字符分割字符串。
3. 取分割后数组的最后一个元素，其长度即为答案。

**关键代码（已改进为更健壮的 `\\s+`）：**

```java
public int lengthOfLastWordBySplit(String s) {
    if (s == null) return 0;
    s = s.trim();
    if (s.isEmpty()) {
        return 0;
    }
    String[] arr = s.split("\\s+");
    return arr[arr.length - 1].length();
}
```

- **时间复杂度：** `O(n)`（trim + split 内部仍需线性扫描）。
- **空间复杂度：** `O(n)`，split 会创建字符串数组及多个子串对象。
- **优点：** 代码极短、语义直观、利用语言内置能力快速实现。
- **缺点：** 额外空间开销大；若使用 `split("\\s")` 而非 `split("\\s+")`，多个连续空格会产生空字符串，容易引入 bug（原代码即为此问题）。

## 四、解法对比表

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 | 推荐指数 |
|------|----------|------|------|------|------|----------|
| 一、从后遍历 | 倒序跳空格 + 计数 | `O(n)` | `O(1)` | 空间最优、无对象分配、通用 | 需处理边界 | ★★★★★ |
| 二、trim + split | 分割后取最后一段 | `O(n)` | `O(n)` | 实现最短、易读 | 空间浪费、split 细节易错 | ★★☆☆☆ |

**推荐：** 面试和生产环境优先使用解法一。解法二仅适合快速原型或对空间不敏感的场景。

## 五、易错点总结

1. **结尾空格必须先跳过**：如果直接从末尾计数，会把空格也算进去。
2. **多个连续空格**：使用 `split("\\s")`（单空格）在单词间有多个空格时会产生 `""` 空串，导致 `arr[arr.length-1]` 可能出错。应使用 `split("\\s+")`。
3. **全为空格的字符串**：`trim()` 后变空串，必须特判返回 0（虽题目保证至少有一个单词，但健壮性代码应考虑）。
4. **没有空格的情况**：整个字符串就是一个单词，应返回其长度。
5. **单个字符 + 空格**：如 `"a "`、`" a"`，经过正确处理后都应返回 1。
6. **不要使用 `substring` 再取 length**：多余操作，解法一直接计数即可。
