# LeetCode 21. 合并两个有序链表（Merge Two Sorted Lists）解题文档

## 一、题目描述

将两个**升序**链表合并为一个新的**升序**链表并返回。新链表是通过拼接给定的两个链表的所有结点组成的。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `l1 = [1,2,4], l2 = [1,3,4]` | `[1,1,2,3,4,4]` | 交替挑较小的头 |
| `l1 = [], l2 = []` | `[]` | 两个空链表 |
| `l1 = [], l2 = [0]` | `[0]` | 一个为空，直接接另一个 |

**提示：**
- 两个链表的结点数目范围是 `[0, 50]`
- `-100 <= Node.val <= 100`
- `l1` 和 `l2` 均按**非递减**顺序排列

**链接：** https://leetcode.cn/problems/merge-two-sorted-lists/

---

## 二、核心观察 / 关键点

1. **归并排序的「合并」步**：两条都已升序，每次比较两个头结点，把**较小的**接到结果尾部，该指针后移，重复直到一条走完。
2. **哑结点（dummy）简化拼接**：新建 `dummy`，用 `cur` 指针一路往后接；最后返回 `dummy.next`，省去「第一个结点单独处理」的特判。
3. **剩余直接整段接上**：当一条链表走完，另一条剩下的部分本身就有序，直接 `cur.next = 剩余` 一次接完，不用逐个搬。
4. **复用原结点、不新建结点**：题目说「拼接所有结点」，所以是改指针把已有结点串起来，O(1) 额外结点空间（迭代法）。

---

## 三、解法

### 解法一：迭代 + 哑结点（推荐，源码 `LeetCode21.mergeTwoLists`）

**思路：**
1. 建 `dummy`，`cur = dummy`。
2. 当 `l1`、`l2` 都非空：谁的 `val` 小就把谁接到 `cur.next`，并把那条链的指针后移，`cur` 也后移。
3. 循环结束后，至多一条还有剩余，直接接到 `cur.next`。
4. 返回 `dummy.next`。

**关键代码片段：**（与源文件 `LeetCode21.java` 同思路，精简版）

```java
ListNode dummy = new ListNode(0);
ListNode cur = dummy;
while (l1 != null && l2 != null) {
    if (l1.val <= l2.val) {     // 取较小的头接上
        cur.next = l1;
        l1 = l1.next;
    } else {
        cur.next = l2;
        l2 = l2.next;
    }
    cur = cur.next;
}
cur.next = (l1 != null) ? l1 : l2;  // 剩余整段接上
return dummy.next;
```

- **时间复杂度：** O(m + n)，每个结点访问一次。
- **空间复杂度：** O(1)，只用了几个指针（不含输入与结果）。
- **优点：** 无递归、无额外栈开销，空间最省。
- **缺点：** 需要哑结点和指针操作，初学者要想清 `cur` 的推进。

> 说明：仓库原始 `LeetCode21.mergeTwoLists` 对 `l1 == null`、`l2 == null` 做了若干前置特判，逻辑等价但略冗长；上面的精简版用 `while` + 末尾「剩余整段接上」即可自然覆盖空链表情形。

---

### 解法二：递归

**思路：** 把「合并」递归定义——较小的头结点 + 「合并剩余两条」的结果。

1. 任一为空，返回另一条。
2. 若 `l1.val <= l2.val`：`l1.next = merge(l1.next, l2)`，返回 `l1`；
3. 否则：`l2.next = merge(l1, l2.next)`，返回 `l2`。

**关键代码片段：**（与 `MergeTwoListsTest.mergeRecursive` 一致）

```java
if (l1 == null) return l2;
if (l2 == null) return l1;
if (l1.val <= l2.val) {
    l1.next = mergeRecursive(l1.next, l2);
    return l1;
} else {
    l2.next = mergeRecursive(l1, l2.next);
    return l2;
}
```

- **时间复杂度：** O(m + n)。
- **空间复杂度：** O(m + n)，递归栈深度等于总结点数。
- **优点：** 代码极简洁，最贴合「有序合并」的递归定义。
- **缺点：** 链表很长时递归栈可能过深（本题 ≤100 结点无忧，但工程上要警惕栈溢出）。

---

## 四、解法对比表

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、迭代 + 哑结点 | 逐个取较小头，剩余整段接 | O(m+n) | O(1) | 省空间、无栈风险 | 指针操作要细心 |
| 二、递归 | 较小头 + 合并剩余 | O(m+n) | O(m+n) | 代码最简洁 | 递归栈可能深 |

---

## 五、易错点总结

1. **务必用哑结点**：否则要单独判断「结果链表的第一个结点是谁」，代码更乱。
2. **稳定性用 `<=` 而非 `<`**：相等时优先取 `l1`，能保持相对顺序稳定（本题对答案无影响，但是好习惯）。
3. **剩余整段接上别漏**：循环退出后还有一条非空，要 `cur.next = l1 != null ? l1 : l2`；漏掉会丢结点。
4. **递归的终止条件**：必须先判 `l1 == null` / `l2 == null` 返回另一条，否则会对 `null` 取 `.val` 抛空指针。
5. **是拼接不是复制**：直接改 `next` 指针复用原结点即可，不需要 `new` 新结点。
