# LeetCode 7. 整数反转

> https://leetcode.cn/problems/reverse-integer/

## 一、问题描述

给你一个 **32 位的有符号整数** `x`，返回将 `x` 中的数字部分**反转**后的结果。

如果反转后的整数超过 32 位有符号整数的范围 `[-2^31, 2^31 - 1]`，就返回 `0`。

**假设环境不允许存储 64 位整数（有符号或无符号）。** —— 这条限制是本题的真正难点，意味着不能简单地用 `long` 兜住溢出。

| 输入 | 输出 | 说明 |
| --- | --- | --- |
| `x = 123` | `321` | 普通反转 |
| `x = -123` | `-321` | 负号保留 |
| `x = 120` | `21` | 末尾的 0 反转后丢弃 |
| `x = 0` | `0` | 原样返回 |
| `x = 1534236469` | `0` | 反转后超出 int 范围 |

提示：`-2^31 <= x <= 2^31 - 1`，即 `-2147483648 <= x <= 2147483647`。

## 二、核心思想：弹出末位 + 推入新末位

反转一个整数，本质是**反复把末位摘下来，接到结果的末尾**：

- **弹出末位**：`digit = x % 10`，再 `x /= 10` 去掉这一位；
- **推入末位**：`result = result * 10 + digit`，把刚摘下的位接到结果末尾。

以 `x = 123` 为例：

| 步骤 | x | digit = x % 10 | result 更新 | result |
| --- | --- | --- | --- | --- |
| 初始 | 123 | - | - | 0 |
| 1 | 12 | 3 | `0*10 + 3` | 3 |
| 2 | 1 | 2 | `3*10 + 2` | 32 |
| 3 | 0 | 1 | `32*10 + 1` | 321 |

`x` 变为 0，循环结束，结果 `321`。

## 三、负数怎么办？不用单独记符号

Java（以及 C/C++）的整数取模/取整是**向零取整**，对负数同样成立：

```
-123 % 10 == -3      -123 / 10 == -12
-120 % 10 == 0       -120 / 10 == -12
```

第二个例子说明：负数末尾的 0 取模得到的就是 `0`，和正数一样会被自然丢弃（`result = result*10 + 0` 不改变结果）。

也就是说负数取出的 `digit` 本身就是负的，`result` 全程保持为负，反转后自然带上负号。
所以**不需要**先取绝对值、再单独记录符号——直接对原值循环即可，代码更简洁，也避免了 `Math.abs(Integer.MIN_VALUE)` 溢出的坑。

以 `x = -123` 为例，全程 `result` 保持为负，符号自然带出：

| 步骤 | x | digit = x % 10 | result 更新 | result |
| --- | --- | --- | --- | --- |
| 初始 | -123 | - | - | 0 |
| 1 | -12 | -3 | `0*10 + (-3)` | -3 |
| 2 | -1 | -2 | `-3*10 + (-2)` | -32 |
| 3 | 0 | -1 | `-32*10 + (-1)` | -321 |

以 `x = -120` 为例，首次 `digit = 0`，前导 0 被自然丢弃：

| 步骤 | x | digit = x % 10 | result 更新 | result |
| --- | --- | --- | --- | --- |
| 初始 | -120 | - | - | 0 |
| 1 | -12 | 0 | `0*10 + 0` | 0 |
| 2 | -1 | -2 | `0*10 + (-2)` | -2 |
| 3 | 0 | -1 | `-2*10 + (-1)` | -21 |

> 末尾为 0 的情况（如 `120`）也被自然处理：第一次 `digit = 0`，`result = 0*10+0 = 0`，相当于前导 0 被丢弃。

## 四、难点：在 int 范围内提前判断溢出

题目限制不能用 64 位整数，所以必须在**做乘法之前**就判断 `result * 10 + digit` 会不会越界。
关键常量：

```
Integer.MAX_VALUE =  2147483647   // 末位是 7
Integer.MIN_VALUE = -2147483648   // 末位是 8
```

在执行 `result = result * 10 + digit` 之前判断：

**正向溢出**（`result > 0`）：
- 若 `result > MAX / 10`（即 `> 214748364`），无论 `digit` 是几，乘 10 必然溢出；
- 若 `result == MAX / 10`，则只有当 `digit > 7` 时才溢出（因为 MAX 末位是 7）。

**负向溢出**（`result < 0`）：
- 若 `result < MIN / 10`（即 `< -214748364`），乘 10 必然溢出；
- 若 `result == MIN / 10`，则只有当 `digit < -8` 时才溢出（因为 MIN 末位是 8）。

只要命中以上任一条件，立即返回 `0`。

> 为什么是"之前"判断而不是"之后"？因为一旦 `result * 10` 已经溢出，结果就已经是错的脏值，
> 再去比较已经无意义——这正是"不允许用 64 位"逼着我们做的**事前检查**。

## 五、代码

```java
public int reverse(int x) {
    int result = 0;
    while (x != 0) {
        int digit = x % 10; // 取出末位（负数时 digit 为负）
        x /= 10;            // 去掉末位

        // 推入前判断正向溢出：result*10 + digit > Integer.MAX_VALUE
        if (result > Integer.MAX_VALUE / 10
                || (result == Integer.MAX_VALUE / 10 && digit > 7)) {
            return 0;
        }
        // 推入前判断负向溢出：result*10 + digit < Integer.MIN_VALUE
        if (result < Integer.MIN_VALUE / 10
                || (result == Integer.MIN_VALUE / 10 && digit < -8)) {
            return 0;
        }

        result = result * 10 + digit;
    }
    return result;
}
```

## 六、复杂度分析

- **时间复杂度** `O(log|x|)`：循环次数等于 `x` 的十进制位数，最多约 10 次。
- **空间复杂度** `O(1)`：只用了常数个变量。

## 七、其他解法对比

| 解法 | 是否满足"禁用 64 位" | 特点 |
| --- | --- | --- |
| **int 内提前判溢出（本文）** | ✅ 满足 | 标准答案，事前检查，无需 long |
| 借助 `long` 暂存再比较 | ❌ 不满足题目限制 | 写法最简单，用 `long reversed` 兜住溢出后再判断是否越界；面试时若被强调限制则不可用 |
| 转成字符串反转 | ✅（但绕） | 处理符号、前导 0、再 `parse` 时仍要捕获溢出，反而更繁琐 |


## 八、小结

整数反转的套路：

1. **弹出 + 推入**：`x % 10` 取末位、`x / 10` 去末位，`result * 10 + digit` 接到结果末尾；
2. **负数免特判**：取模/取整向零取整，负号自然保留，不必单独记符号；
3. **事前判溢出**：在乘 10 之前用 `MAX/10`、`MIN/10` 和末位 `7 / 8` 做边界判断，越界返回 `0`；
4. 末尾 0 会被自然丢弃，无需额外处理。
