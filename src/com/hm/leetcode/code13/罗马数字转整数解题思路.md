# LeetCode 13. 罗马数字转整数（Roman to Integer）解题文档

## 一、题目描述

罗马数字包含七种字符：`I`、`V`、`X`、`L`、`C`、`D`、`M`，对应数值：

| 字符 | I | V | X | L | C | D | M |
|------|---|---|---|---|---|---|---|
| 数值 | 1 | 5 | 10 | 50 | 100 | 500 | 1000 |

通常小数字在大数字右边（做加法）。但有 **6 种减法特例**：小数字在大数字左边表示相减（`IV=4`、`IX=9`、`XL=40`、`XC=90`、`CD=400`、`CM=900`）。

给定一个罗马数字，将其转换成整数。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `III` | 3 | 1 + 1 + 1 |
| `IV` | 4 | 5 - 1 |
| `IX` | 9 | 10 - 1 |
| `LVIII` | 58 | L(50) + V(5) + III(3) |
| `MCMXCIV` | 1994 | M(1000) + CM(900) + XC(90) + IV(4) |

**链接：** https://leetcode.cn/problems/roman-to-integer/

---

## 二、核心观察

把罗马数字转整数的本质规律提炼成一句话：

> **遍历每个字符，若它比右边相邻字符小，则减去它；否则加上它。**

原因：减法特例（如 `IV`）的唯一特征就是「**小的数字在大的数字左边**」。只要发现「当前字符 < 右边字符」，就说明它是被减项，取负即可；其余情况都做加法。这样**无需单独枚举 6 种特例**。

由这条规律，可以从两个方向遍历：
- **从右往左**：只需记住「上一个（右侧）值」，比较 `当前 vs prev`。
- **从左往右**：需要「向后看一位」，比较 `当前 vs 下一个`。

---

## 三、解法

### 解法一：从右往左遍历（推荐）

**思路：**
1. 从字符串末尾向前遍历，用 `prev` 记录右侧（上一个）字符的值。
2. 当前值 `curr >= prev` → 累加；`curr < prev` → 累减（减法特例）。
3. 每轮更新 `prev = curr`。

```java
public static int romanToInt(String s) {
    int result = 0;
    int prev = 0;                       // 右侧（上一个）字符的值
    for (int i = s.length() - 1; i >= 0; i--) {
        int curr = valueOf(s.charAt(i));
        if (curr >= prev) {
            result += curr;             // 正常累加
        } else {
            result -= curr;             // 减法特例，如 IV 中的 I
        }
        prev = curr;
    }
    return result;
}
```

- **时间复杂度：** O(n)，n 为字符串长度
- **空间复杂度：** O(1)
- **优点：** 只需一个 `prev` 变量，逻辑最简洁
- **缺点：** 从右往左遍历，直觉上稍不自然

---

### 解法二：从左往右遍历（向后看一位）

**思路：**
1. 从左到右遍历，每个字符与**右边相邻**字符比较。
2. 若 `当前值 < 右边值`（且不是最后一位）→ 减去当前值；否则加上。

```java
public static int romanToInt2(String s) {
    int result = 0, n = s.length();
    for (int i = 0; i < n; i++) {
        int curr = valueOf(s.charAt(i));
        if (i < n - 1 && curr < valueOf(s.charAt(i + 1))) {
            result -= curr;             // 减法形式，如 IV 的 I
        } else {
            result += curr;
        }
    }
    return result;
}
```

- **时间复杂度：** O(n)
- **空间复杂度：** O(1)
- **优点：** 从左到右符合阅读直觉
- **缺点：** 需判断越界（最后一位）、每轮多取一次右邻值

---

### 字符取值：用 switch 代替 HashMap

```java
private static int valueOf(char c) {
    switch (c) {
        case 'I': return 1;
        case 'V': return 5;
        case 'X': return 10;
        case 'L': return 50;
        case 'C': return 100;
        case 'D': return 500;
        case 'M': return 1000;
        default:  return 0;
    }
}
```

字符集固定只有 7 个，`switch` 避免了 `HashMap` 的装箱（`Integer`）与哈希计算开销，常数更小。

## 四、解法对比

| 解法 | 遍历方向 | 比较对象 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|----------|------|------|------|------|
| 一、从右往左 | ← | 当前 vs 上一个 | O(n) | O(1) | 只用一个 prev，最简洁 | 方向不直觉 |
| 二、从左往右 | → | 当前 vs 下一个 | O(n) | O(1) | 符合阅读直觉 | 需判越界、多取一次 |

> 两种本质相同，都是「小数在大数左边即取负」。面试任选其一，**解法一**代码最短。

---

## 五、易错点总结

1. **减法判断方向**：核心是「比右边小就减」。解法一比较的是 `prev`（右侧），解法二比较的是 `i+1`（右侧），方向不能搞反。
2. **边界越界**：解法二要先判断 `i < n - 1`，否则最后一位访问 `s.charAt(i+1)` 越界。
3. **相等用 `>=`**：解法一里连续相同字符（如 `III`）应走加法，所以条件是 `curr >= prev` 而非 `curr > prev`。
4. **题目保证输入合法**：无需校验非法罗马数字，可省去额外判断。
