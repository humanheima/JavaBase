# LeetCode 20. 有效的括号（Valid Parentheses）解题文档

## 一、题目描述

给定一个只包括 `'('`、`')'`、`'{'`、`'}'`、`'['`、`']'` 的字符串 `s`，判断字符串是否**有效**。有效字符串需满足：

1. 左括号必须用**相同类型**的右括号闭合；
2. 左括号必须以**正确的顺序**闭合；
3. 每个右括号都有一个对应的相同类型的左括号。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `"()"` | `true` | 一对匹配 |
| `"()[]{}"` | `true` | 三对依次匹配 |
| `"(]"` | `false` | 类型不匹配 |
| `"([)]"` | `false` | 顺序错误（交叉闭合） |
| `"{[]}"` | `true` | 正确嵌套 |

**提示：**
- `1 <= s.length <= 10^4`
- `s` 仅由括号 `'()[]{}'` 组成

**链接：** https://leetcode.cn/problems/valid-parentheses/

---

## 二、核心观察 / 关键点

1. **典型的「栈」结构**：括号要求**后开的先闭合**——最近压入的左括号必须最先被匹配，这正是「后进先出（LIFO）」，天然用栈。
2. **遇左括号压栈，遇右括号弹栈比对**：右括号来时，栈顶必须是与之**同类型**的左括号。
3. **三种 false 情形**要全覆盖：
   - 右括号来时**栈为空**（没有左括号可配）→ 如 `")"`、`"]["`；
   - 弹出的栈顶与当前右括号**类型不符** → 如 `"(]"`、`"([)]"`；
   - 遍历结束后**栈非空**（还有左括号没闭合）→ 如 `"("`。
4. **长度为奇数必定无效**：括号要两两闭合，奇数长度直接 `false`，可作小剪枝。

## 三、解法

### 解法一：栈 + 辅助判断函数（源码 `LeetCode20.isValid`）

**思路：**
1. 遍历字符：左括号 `( { [` 直接压栈。
2. 右括号 `) } ]`：若栈空返回 `false`；否则弹出栈顶，用 `isMatchingPair` 判断是否同类型，不符返回 `false`。
3. 遍历结束，栈空则有效。

**关键代码片段：**（与源文件 `LeetCode20.java` 一致）

```java
Stack<Character> stack = new Stack<>();
for (char c : s.toCharArray()) {
    if (c == '(' || c == '{' || c == '[') {
        stack.push(c);                       // 左括号压栈
    } else if (c == ')' || c == '}' || c == ']') {
        if (stack.isEmpty()) return false;   // 没有左括号可配
        char top = stack.pop();
        if (!isMatchingPair(top, c)) return false; // 类型不符
    }
}
return stack.isEmpty();                       // 还有未闭合的左括号 -> false
```

- **时间复杂度：** O(n)，每个字符进出栈各一次。
- **空间复杂度：** O(n)，最坏全是左括号时栈深 n。
- **优点：** 逻辑直白，匹配判断集中在一个辅助函数里，易读。
- **缺点：** `isMatchingPair` 用多个 `||` 列举，扩展新括号类型时要改函数。

---

### 解法二：HashMap 映射 + 栈（更易扩展）

**思路：** 用 `Map<右括号, 左括号>` 存匹配关系。遍历时：
- 当前字符是右括号（在 map 的 key 里）：栈空或弹出的栈顶 `!=` map 中对应的左括号，则 `false`；
- 否则是左括号，压栈。

**关键代码片段：**（与 `ValidParenthesesTest.isValidByMap` 一致）

```java
if (s.length() % 2 != 0) return false;        // 奇数长度剪枝
Map<Character, Character> pairs = new HashMap<>();
pairs.put(')', '('); pairs.put(']', '['); pairs.put('}', '{');

Deque<Character> stack = new ArrayDeque<>();
for (char c : s.toCharArray()) {
    if (pairs.containsKey(c)) {                // 右括号
        if (stack.isEmpty() || stack.pop() != pairs.get(c)) return false;
    } else {                                   // 左括号
        stack.push(c);
    }
}
return stack.isEmpty();
```

- **时间复杂度：** O(n)。
- **空间复杂度：** O(n)。
- **优点：** 匹配关系集中在 map 里，新增括号类型只改 map；用 `ArrayDeque` 当栈比 `Stack` 更快（`Stack` 继承自 `Vector`，方法带 `synchronized`）。
- **缺点：** 需要额外建一个 map。

## 五、易错点总结

1. **右括号时先判栈空**：必须先 `stack.isEmpty()` 再 `pop()`，否则空栈 `pop` 会抛异常（如输入 `")"`）。
2. **结尾别忘判栈空**：遍历完要 `return stack.isEmpty()`，漏掉会把 `"("` 这种「只开不闭」误判为有效。
3. **类型必须同种**：`(]` 这类左右数量对得上但类型不符，要靠匹配判断拦截，不能只数数量。
4. **顺序（交叉）问题**：`([)]` 靠栈的 LIFO 天然处理——`)` 来时栈顶是 `[` 而非 `(`，直接 false。
5. **栈的选择**：工程中优先用 `ArrayDeque` 而非 `java.util.Stack`（后者同步开销大、是过时的 `Vector` 子类）。
6. **奇数长度剪枝**：可选优化，能提前否掉一半无效输入。
