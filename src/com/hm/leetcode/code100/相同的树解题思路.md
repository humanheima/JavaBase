# LeetCode 100. 相同的树（Same Tree）解题文档

## 一、题目描述

给你两棵二叉树的根节点 `p` 和 `q`，编写一个函数来检验这两棵树是否**相同**。

如果两棵树在**结构上相同**，并且**节点具有相同的值**，则认为它们是相同的。

**示例：**

| p | q | 输出 | 说明 |
|---|---|------|------|
| `[1,2,3]` | `[1,2,3]` | `true` | 结构、值都一致 |
| `[1,2]` | `[1,null,2]` | `false` | 一个左孩子、一个右孩子，结构不同 |
| `[1,2,1]` | `[1,1,2]` | `false` | 结构相同但对应节点值不同 |

**链接：** https://leetcode.cn/problems/same-tree/

---

## 二、核心观察

1. 「相同」要同时满足两点：**结构相同** + **对应位置的值相同**，缺一不可。
2. 判断可以递归分解为：**根相同** 且 **左子树相同** 且 **右子树相同**——这是天然的子问题结构。
3. 比较的关键在于成对处理 `(p, q)`：三类空值情况要分清——
   - 都为空 → 这一对相同（递归出口 / 迭代 continue）；
   - 恰好一个为空 → 结构不同，直接 `false`；
   - 都非空 → 比较值，再向下递归 / 入队。

---

## 三、两种解法

### 解法一：递归（DFS）

**思路：**
1. 出口：`p` 和 `q` 都为空 → `true`。
2. 一个为空一个非空 → 结构不同 → `false`。
3. 都非空但值不同 → `false`。
4. 否则递归比较「左子树相同」**且**「右子树相同」。

```java
public boolean isSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) return true;
    if (p == null || q == null) return false;
    if (p.val != q.val) return false;
    return isSameTree(p.left, q.left) && isSameTree(p.right, q.right);
}
```

- **时间复杂度：** O(n)，n 为较小树的节点数，每对节点最多访问一次
- **空间复杂度：** O(n)，递归栈最坏（链状树）为 O(n)，平衡时为 O(log n)
- **优点：** 代码最短，与「相同」定义一一对应，可读性极佳
- **缺点：** 依赖系统调用栈，极深的树有栈溢出风险

---

### 解法二：迭代（BFS，队列成对比较）

**思路：**
1. 用队列把 `(p, q)` **成对**入队。
2. 每次成对出队两个节点 `node1`、`node2`：
   - 都为空 → `continue`（这一对没问题）；
   - 恰好一个为空 → `return false`；
   - 值不同 → `return false`。
3. 把两节点的左孩子、右孩子**按对**入队，继续循环。
4. 队列清空仍未发现差异 → `return true`。

```java
public boolean isSameTreeIterative(TreeNode p, TreeNode q) {
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(p);
    queue.offer(q);

    while (!queue.isEmpty()) {
        TreeNode node1 = queue.poll();
        TreeNode node2 = queue.poll();

        if (node1 == null && node2 == null) continue;   // 都空，跳过
        if (node1 == null || node2 == null) return false; // 一空一非空
        if (node1.val != node2.val) return false;         // 值不同

        queue.offer(node1.left);   // 子节点成对入队
        queue.offer(node2.left);
        queue.offer(node1.right);
        queue.offer(node2.right);
    }
    return true;
}
```

- **时间复杂度：** O(n)
- **空间复杂度：** O(n)，队列最坏存放一层的所有节点
- **优点：** 不依赖系统栈，避免深树栈溢出；显式控制遍历过程
- **缺点：** 需小心维护「成对入队、成对出队」的不变式，否则比较错位

---

## 四、两种解法对比

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、递归 DFS | 根同 + 左同 + 右同 | O(n) | O(n) 栈 | 最简洁、贴合定义 | 深树栈溢出风险 |
| 二、迭代 BFS | 队列成对比较 | O(n) | O(n) 队列 | 不依赖系统栈 | 需维护成对不变式 |

---

## 五、易错点总结

1. **空值三种情况要分清**：都空（相同）、一空一非空（不同）、都非空（继续比值）。漏掉「一空一非空」会在结构不同的树上误判。
2. **判断顺序**：必须先处理空值，再访问 `.val`，否则对 `null` 取 `val` 会空指针。
3. **迭代必须严格成对**：入队和出队都要成对，且 `node1.left` 配 `node2.left`、`node1.right` 配 `node2.right`，顺序错位会把不同位置的节点拿来比较。
4. **空节点也要入队**：迭代解法里子节点即使为 `null` 也照样 `offer`，靠出队时的「都空则 continue」来消化，这样才能检测出结构差异。
