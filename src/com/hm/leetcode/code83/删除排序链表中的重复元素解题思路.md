# LeetCode 83. 删除排序链表中的重复元素（Remove Duplicates from Sorted List）解题文档

## 一、题目描述

给定一个**已排序**的链表的头 `head`，删除所有重复的元素，使每个元素**只出现一次**。返回已排序的链表。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `1->1->2` | `1->2` | 两个 1 合并为一个 |
| `1->1->2->3->3` | `1->2->3` | 1 与 3 各保留一个 |
| `1->1->2->2->5` | `1->2->5` | 每个值只保留首个出现 |

**链接：** https://leetcode.cn/problems/remove-duplicates-from-sorted-list/

---

## 二、核心观察

1. **链表已排序**：所有相等的值一定是**相邻**的，因此判重只需比较当前节点与其后继节点，无需哈希表，无需回看。
2. 与 LeetCode 82（删除全部重复节点、一个不留）不同：本题对重复值**保留一个**，因此不需要 `dummy` 哨兵节点——头节点一定会被保留。
3. 删除操作就是改指针：`current.next = current.next.next`，把重复节点从链上「跳过」。

---

## 三、两种解法

### 解法一：单指针原地遍历（推荐）

**思路：**
1. 边界：链表为空或只有一个节点，直接返回 `head`。
2. 用一个 `current` 指针从头开始遍历。
3. 每一步比较 `current.val` 与 `current.next.val`：
   - **相等**：说明 `current.next` 是重复项，执行 `current.next = current.next.next` 跳过它。**注意此时 `current` 不前移**，因为新的 `current.next` 可能仍与 `current` 相等（如 `1->1->1`）。
   - **不相等**：`current` 前移到 `current.next`。
4. 直到 `current` 或 `current.next` 为空，遍历结束。

```java
public ListNode deleteDuplicates(ListNode head) {
    // 边界情况：空链表或只有一个节点
    if (head == null || head.next == null) {
        return head;
    }

    ListNode current = head;
    while (current != null && current.next != null) {
        if (current.val == current.next.val) {
            // 跳过下一个节点（删除重复元素），current 不前移
            current.next = current.next.next;
        } else {
            // 值不同，移动到下一个节点
            current = current.next;
        }
    }
    return head;
}
```

- **时间复杂度：** O(n)，每个节点访问一次
- **空间复杂度：** O(1)，仅用常数指针
- **优点：** 思路最直接，无需哨兵节点，代码精炼
- **缺点：** 需注意「相等时不前移」这一细节，否则连续多个重复值会漏删

---

### 解法二：哨兵 + 内层 while 跳重复（想复杂了）

**思路：**
1. 建立 `dummy` 哨兵指向 `head`，`pre` 指向哨兵，`cur` 指向当前节点。
2. 当 `cur` 与 `cur.next` 相等时，用内层 `while` 一路向后吞掉所有相同值，再用 `pre.next` 接上后续。
3. 这种写法本是为「LeetCode 82 一个不留」准备的模板，套到本题上属于「杀鸡用牛刀」。

```java
public ListNode deleteDuplicates1(ListNode head) {
    if (head == null) {
        return null;
    }
    ListNode dummy = new ListNode(Integer.MIN_VALUE);
    dummy.next = head;
    ListNode pre = dummy;
    ListNode cur = head;

    while (cur != null) {
        if (cur.next != null && cur.val == cur.next.val) {
            while (cur.next != null && cur.next.val == cur.val) {
                cur = cur.next;
            }
            pre = pre.next;
            pre.next = cur.next;
        }
        pre = cur;
        cur = cur.next;
    }
    return dummy.next;
}
```

- **时间复杂度：** O(n)
- **空间复杂度：** O(1)
- **优点：** 与「删除全部重复」题解模板统一，迁移成本低
- **缺点：** 引入 `dummy`、`pre`、内外双层循环，对本题而言冗余，可读性下降

---

## 四、两种解法对比

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、单指针原地遍历 | 相等则跳过 next，否则前移 | O(n) | O(1) | 精炼、无哨兵 | 需记住「相等不前移」 |
| 二、哨兵 + 内层 while | 哨兵 + 双层循环吞重复 | O(n) | O(1) | 与 82 题模板统一 | 冗余、可读性差 |

---

## 五、易错点总结

1. **相等时不要前移 `current`**：`1->1->1` 中若删完一个就前移，会漏掉后面的重复。删除后应让 `current` 留在原地，继续与新的后继比较。
2. **循环条件要双判空**：`current != null && current.next != null`，否则访问 `current.next.val` 会空指针。
3. **本题不需要 `dummy`**：因为头节点必然保留，加哨兵反而把简单题写复杂（区别于「删除全部重复」需要哨兵处理头节点也可能被删的情况）。
4. **前提是有序**：解法依赖「相同值相邻」。若链表无序，此法失效，需先排序或改用哈希表。
