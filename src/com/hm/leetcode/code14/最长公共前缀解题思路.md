# LeetCode 14. 最长公共前缀（Longest Common Prefix）解题文档

## 一、题目描述

编写一个函数来查找字符串数组中的**最长公共前缀**。如果不存在公共前缀，返回空字符串 `""`。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `["flower","flow","flight"]` | `"fl"` | 三者都以 `fl` 开头 |
| `["dog","racecar","car"]` | `""` | 不存在公共前缀 |
| `["ab","a"]` | `"a"` | 受最短串限制 |

**提示：** `1 <= strs.length <= 200`，`0 <= strs[i].length <= 200`，仅小写字母。

**链接：** https://leetcode.cn/problems/longest-common-prefix/

---

## 二、核心观察

1. **公共前缀的长度 ≤ 最短字符串的长度**：任何一个串提前结束，前缀就到此为止。
2. **前缀只会变短不会变长**：从某个起点出发，一旦某位出现分歧就必须停止。
3. 两种自然的扫描方向：
   - **纵向**：固定列、扫所有行（逐字符比所有串的同一位置）。
   - **横向**：固定行、两两归并（前缀和下一个串求交集，逐步缩短）。

---

## 三、解法

### 解法一：纵向扫描（推荐）

**思路：**
1. 以第一个字符串 `first` 为基准，逐列 `i` 取出字符 `c`。
2. 让其余每个字符串都来比较第 `i` 位：若某串**已到末尾**（`i >= 长度`）或**该位字符不同**，则公共前缀就是 `first[0..i)`，直接返回。
3. 全部走完说明 `first` 本身就是公共前缀。

```java
public static String longestCommonPrefixVertical(String[] strs) {
    if (strs == null || strs.length == 0) return "";
    String first = strs[0];
    for (int i = 0; i < first.length(); i++) {
        char c = first.charAt(i);
        for (int j = 1; j < strs.length; j++) {
            if (i >= strs[j].length() || strs[j].charAt(i) != c) {
                return first.substring(0, i);   // 该位分歧，截断
            }
        }
    }
    return first;
}
```

- **时间复杂度：** O(S)，S 为所有字符总数（最坏全部字符都比较一遍）
- **空间复杂度：** O(1)
- **优点：** 一旦遇到分歧立刻返回，平均很快
- **缺点：** 嵌套循环，下标边界要小心

---

### 解法二：横向扫描

**思路：**
1. 把第一个字符串当作初始前缀 `prefix`。
2. 依次和后面每个字符串求公共前缀，`prefix` 只会越来越短。
3. 一旦 `prefix` 变空，可提前返回 `""`。

```java
public static String longestCommonPrefixHorizontal(String[] strs) {
    if (strs == null || strs.length == 0) return "";
    String prefix = strs[0];
    for (int i = 1; i < strs.length; i++) {
        prefix = commonPrefix(prefix, strs[i]);
        if (prefix.isEmpty()) return "";
    }
    return prefix;
}

private static String commonPrefix(String s1, String s2) {
    int len = Math.min(s1.length(), s2.length());
    int i = 0;
    while (i < len && s1.charAt(i) == s2.charAt(i)) i++;
    return s1.substring(0, i);
}
```

- **时间复杂度：** O(S)
- **空间复杂度：** O(1)
- **优点：** 逻辑直观，把多串问题拆成两两比较
- **缺点：** 最坏情况要遍历完所有串才知道结果

## 四、解法对比

| 解法 | 扫描方式 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、纵向扫描 | 逐列比所有串 | O(S) | O(1) | 遇分歧即停，平均快 | 下标边界易错 |
| 二、横向扫描 | 两两归并前缀 | O(S) | O(1) | 直观、可提前终止 | 最坏遍历全部 |

> 两者最坏复杂度相同（S = 字符总数）。面试推荐**纵向扫描**，遇到分歧能最早退出；横向扫描在思路表达上更直观。
>
> 其他思路：还可对数组按字典序排序后只比较**首尾两个字符串**（最大最小串的公共前缀即为答案），或用**分治 / 二分前缀长度**，但实现更复杂，常规场景上面两种已足够。

---

## 五、易错点总结

1. **空串处理**：数组中含有 `""` 时，公共前缀必为 `""`；纵向扫描靠 `i >= strs[j].length()` 自动处理。
2. **下标越界**：纵向扫描比较前必须先判断 `i >= strs[j].length()`，否则 `charAt(i)` 越界。
3. **substring 边界**：返回 `first.substring(0, i)`，`i` 是第一个不匹配的位置（不含），正好是前缀长度。
4. **单元素数组**：只有一个串时，它本身就是答案，两种解法都能自然覆盖。
