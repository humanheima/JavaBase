# LeetCode 145. 二叉树的后序遍历（Binary Tree Postorder Traversal）解题文档

## 一、题目描述

给定一个二叉树的根节点 `root`，返回它的**后序遍历**结果。

后序遍历的顺序是 **左子树 → 右子树 → 根节点**（左 - 右 - 根）。

**示例：**

```
        1
       / \
      2   3
     / \   \
    4   5   6
```

| 输入 | 输出 | 说明 |
|------|------|------|
| 上图的 `root` | `[4, 5, 2, 6, 3, 1]` | 左子树(4,5,2) → 右子树(6,3) → 根(1) |
| `[]`（空树） | `[]` | 空树返回空列表 |
| `[1]` | `[1]` | 单节点 |

**链接：** https://leetcode.cn/problems/binary-tree-postorder-traversal/

---

## 二、核心观察

1. **后序遍历的本质**是「**左 - 右 - 根**」：必须先完整处理完左右子树，才能处理根节点。这与「先处理子问题再处理当前问题」的递归思想高度吻合。
2. 递归实现最直观，但系统栈深度在最坏情况下（链状树）为 O(n)。
3. **迭代难点**：后序遍历是三种 DFS 遍历中最难用栈模拟的。因为访问根节点必须在左右子树都处理完之后，需要记录「右子树是否已访问」。
4. 常见迭代技巧：
   - 双栈法：利用栈的 LIFO 特性，通过两次反转得到正确顺序（最容易理解）。
   - 单栈 + lastVisited 标记：记录上次访问的节点，判断右子树是否处理完毕。
5. 后序遍历常用于「需要先处理完子节点才能决定父节点行为」的场景（如删除节点、计算目录大小、后缀表达式求值）。

---

## 三、三种解法

### 解法一：递归（最直观推荐）

**思路：**
1. 递归边界：节点为空直接返回。
2. 先递归遍历左子树。
3. 再递归遍历右子树。
4. 最后访问当前根节点（加入结果列表）。

```java
public List<Integer> postorderTraversal(TreeNode root) {
    result.clear();
    postorder(root);
    return result;
}

private void postorder(TreeNode node) {
    if (node == null) {
        return;
    }
    postorder(node.left);   // 左
    postorder(node.right);  // 右
    result.add(node.val);   // 根
}
```

- **时间复杂度：** O(n)，每个节点访问一次
- **空间复杂度：** O(n)，递归调用栈，最坏 O(n)（链状），平均 O(log n)（平衡树）
- **优点：** 代码最短、最符合后序定义，可读性极强
- **缺点：** 依赖系统调用栈，树很深时有栈溢出风险

---

### 解法二：迭代（双栈法）

**思路：**
1. 使用两个栈 `stack1` 和 `stack2`。
2. 将根节点压入 `stack1`。
3. 循环 `stack1` 为空前：
   - 弹出 `stack1` 栈顶节点，压入 `stack2`。
   - **先**将该节点的左孩子压入 `stack1`（如果有）。
   - **再**将该节点的右孩子压入 `stack1`（如果有）。
4. 最后依次弹出 `stack2` 的节点加入结果（由于两次反转，顺序变为 左-右-根）。

```java
public List<Integer> postorderTraversal2(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;

    Stack<TreeNode> stack1 = new Stack<>();
    Stack<TreeNode> stack2 = new Stack<>();
    stack1.push(root);

    while (!stack1.isEmpty()) {
        TreeNode node = stack1.pop();
        stack2.push(node);

        if (node.left != null) stack1.push(node.left);
        if (node.right != null) stack1.push(node.right);
    }

    while (!stack2.isEmpty()) {
        result.add(stack2.pop().val);
    }
    return result;
}
```

- **时间复杂度：** O(n)
- **空间复杂度：** O(n)，需要两个栈
- **优点：** 逻辑清晰、容易记忆和调试，不易出错
- **缺点：** 额外使用一个栈，空间占用是单栈法的两倍

---

### 解法三：迭代（单栈 + lastVisited 标记）

**思路：**
1. 使用一个栈 + 一个 `lastVisited` 指针记录「上一次真正访问（弹出并加入结果）的节点」。
2. 一直向左走到尽头并把沿途节点压栈。
3. 查看栈顶节点 `peekNode`：
   - 若 `peekNode` 没有右子树，或其右子树已经被访问过（`peekNode.right == lastVisited`），说明左右子树都处理完了 → 弹出并访问该节点，更新 `lastVisited`。
   - 否则，说明右子树还没处理 → 将 `curr` 指向右子树，继续循环。
4. 重复直到栈空且当前节点为空。

```java
public List<Integer> postorderTraversal3(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;

    Stack<TreeNode> stack = new Stack<>();
    TreeNode curr = root;
    TreeNode lastVisited = null;

    while (curr != null || !stack.isEmpty()) {
        while (curr != null) {
            stack.push(curr);
            curr = curr.left;
        }

        TreeNode peekNode = stack.peek();
        if (peekNode.right == null || peekNode.right == lastVisited) {
            TreeNode node = stack.pop();
            result.add(node.val);
            lastVisited = node;
            curr = null;
        } else {
            curr = peekNode.right;
        }
    }
    return result;
}
```

- **时间复杂度：** O(n)
- **空间复杂度：** O(n)（仅一个显式栈）
- **优点：** 只用一个栈，空间利用率更高；是面试中常考的「硬核」迭代写法
- **缺点：** 逻辑相对复杂，`lastVisited` 的判断条件容易写错

## 四、三种解法对比

| 解法 | 核心思路 | 时间 | 空间 | 优点 | 缺点 |
|------|----------|------|------|------|------|
| 一、递归 | 左 → 右 → 根 直接递归 | O(n) | O(n) 栈 | 最简洁、直观 | 依赖系统栈，深树风险 |
| 二、双栈迭代 | 两次栈反转模拟访问顺序 | O(n) | O(n)（两个栈） | 思路最清晰，代码好写 | 空间多一倍 |
| 三、单栈 + lastVisited | 左链压栈 + 右子树访问标记 | O(n) | O(n)（一个栈） | 空间最优，经典面试写法 | 状态判断较复杂，易错 |

---

## 五、易错点总结

1. **访问顺序极易搞反**：后序是「左-右-根」，前序是「根-左-右」，中序是「左-根-右」。写递归时 `result.add` 的位置最容易放错。
2. **单栈法 `lastVisited` 判断**：
   - 必须同时判断 `right == null` 或 `right == lastVisited`。
   - 更新 `lastVisited` 必须在弹出节点**之后**。
   - 把 `curr = null` 很重要，否则会重复向左。
3. **双栈法压栈顺序**：必须**先左后右**压 `stack1`，这样 `stack2` 反转后才是左-右-根（与前序正好相反）。
4. **空节点处理**：所有迭代写法入口都要先判 `root == null` 返回空列表。
5. **与层次遍历混淆**：后序遍历**不是** BFS，不能用普通队列直接实现（队列适合层序遍历）。
