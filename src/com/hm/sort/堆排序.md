# 堆排序算法解析与 Java 实现

## 一、算法概述

堆排序（Heap Sort）是利用 **堆** 这种数据结构设计的一种选择排序。它的核心思想是：

1. 把无序数组构造成一个 **大顶堆**（堆顶即当前最大值）；
2. 把堆顶元素与末尾元素交换，最大值就被"沉"到了数组末尾（归位）；
3. 对剩下的部分重新调整成堆，重复第 2 步，直到整个数组有序。

可以把它理解为"加强版的选择排序"：选择排序每轮线性扫描找最大值是 `O(n)`，而堆排序借助堆结构把"找最大值"降到 `O(log n)`。

---

## 二、关键概念：堆与数组的映射

堆在逻辑上是一棵 **完全二叉树**，但物理上用 **数组** 存储，靠下标换算父子关系（下标从 0 开始）：

```
       9(0)
      /    \
   12(1)   17(2)
   /   \    /
 30(3) 50(4) 20(5)
```

| 关系 | 公式 |
|------|------|
| 结点 `i` 的左孩子 | `2*i + 1` |
| 结点 `i` 的右孩子 | `2*i + 2` |
| 结点 `i` 的父亲 | `(i - 1) / 2` |
| 最后一个非叶子结点 | `len/2 - 1`（`len` 为元素个数） |

- **大顶堆（Max-Heap）**：每个父结点 ≥ 它的孩子 → 用于 **升序** 排序。
- **小顶堆（Min-Heap）**：每个父结点 ≤ 它的孩子 → 用于 **降序** 排序。

> 升序为什么用大顶堆？因为每轮把最大值换到末尾，末尾从后往前依次被填上越来越小的数，最终整体升序。

---

## 三、算法特点

- **时间复杂度**：
    - 建堆：`O(n)`
    - 排序（n 次取堆顶 + 调整）：`O(n log n)`
    - 综合（最好 / 平均 / 最坏均为）：**`O(n log n)`**
- **空间复杂度**：`O(1)`（原地排序，递归实现的栈深为 `O(log n)`）
- **稳定性**：**不稳定**（交换跨距离，相同元素相对次序可能被打乱）

---

## 四、两个核心步骤

### 步骤 1：建堆（heapify 全部）

从 **最后一个非叶子结点** `len/2 - 1` 开始，从后往前对每个结点执行"下沉"调整。叶子结点本身就满足堆性质，无需处理。

### 步骤 2：排序（反复取堆顶）

每次把堆顶 `arr[0]`（当前最大）与当前未排序区间的末尾交换，然后把未排序长度减 1，再对新的堆顶做一次"下沉"调整，恢复大顶堆。重复直到区间长度为 0。

### 核心操作：下沉（maxHeapify）

让一个"可能不满足堆性质"的父结点，与它 **较大的那个孩子** 比较；若孩子更大就交换，然后继续在新位置向下检查，直到没有孩子比它大或到达叶子。

```
父结点比孩子小 → 和较大的孩子交换 → 在新位置继续下沉 → 直到稳定
```

## 五、Java 实现

> 对应源文件：`HeapSort.java`（递归实现下沉，含大顶堆 `maxHeapify` 与可选的小顶堆 `minHeapify`）。

```java
public class HeapSort {

    private int[] arr;

    public HeapSort(int[] arr) {
        this.arr = arr;
    }

    public void sort() {
        // 第一步：建堆。从最后一个非叶子结点 len/2-1 起，从后往前依次下沉
        int len = arr.length - 1;
        int beginIndex = len / 2 - 1;
        for (int i = beginIndex; i >= 0; i--) {
            maxHeapify(i, len);
        }

        // 第二步：排序。每次把堆顶(最大值)换到末尾，再对剩余部分重新下沉堆顶
        for (int i = len; i > 0; i--) {
            swap(0, i);
            maxHeapify(0, i - 1);
        }
    }

    private void swap(int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }

    /**
     * 下沉调整：让 currentIndex 处的结点满足大顶堆性质
     * @param currentIndex 待调整结点
     * @param len          当前未排序区间的最大下标
     */
    private void maxHeapify(int currentIndex, int len) {
        int leftIndex = currentIndex * 2 + 1;   // 左孩子
        int rightIndex = leftIndex + 1;          // 右孩子
        int maxIndex = leftIndex;                // 默认较大孩子为左孩子
        if (leftIndex > len) return;             // 没有孩子（叶子），结束
        if (rightIndex <= len && arr[rightIndex] > arr[leftIndex]) {
            maxIndex = rightIndex;               // 右孩子更大
        }
        if (arr[maxIndex] > arr[currentIndex]) {
            swap(maxIndex, currentIndex);        // 父 < 较大孩子，交换
            maxHeapify(maxIndex, len);           // 在新位置继续下沉
        }
    }
}
```

## 六、使用示例

```java
public static void main(String[] args) {
    int[] arr = new int[]{9, 12, 17, 30, 50, 20, 60, 65, 4, 49};
    new HeapSort(arr).sort();
    System.out.println(Arrays.toString(arr));
}
```

**实测运行输出（升序）：**

```
[4, 9, 12, 17, 20, 30, 49, 50, 60, 65]
```

> 若要 **降序**，把 `sort()` 中的两处 `maxHeapify` 换成源码里已写好的 `minHeapify`（建小顶堆，每轮把最小值换到末尾）即可。

## 七、过程演示（建堆阶段）

以数组 `[4, 10, 3, 5, 1]` 为例，把它建成大顶堆：

```
初始完全二叉树：
        4(0)
       /    \
    10(1)    3(2)
    /   \
  5(3)  1(4)

最后一个非叶子结点 = 5/2 - 1 = 1，从 i=1 开始往前下沉：

i=1：结点 10，孩子 5、1 都比它小 → 不动
        4
       / \
      10   3
     /  \
    5    1

i=0：结点 4，较大孩子是 10 → 交换 4 和 10
        10
       /  \
      4    3
     / \
    5   1
   继续下沉 4（现在在 index=1），较大孩子是 5 → 交换
        10
       /  \
      5    3
     / \
    4   1

建堆完成：[10, 5, 3, 4, 1]，堆顶 10 即最大值
```

随后的排序阶段：把堆顶 `10` 与末尾交换并归位 → 对剩余 `[1,5,3,4]` 重新下沉堆顶 → 取出新堆顶 `5` …… 如此反复，最终得到升序数组。

## 八、易错点总结

1. **建堆起点**：从 `len/2 - 1`（最后一个非叶子结点）开始，**从后往前**，不能从 0 开始正向建堆。
2. **下沉要和"较大的孩子"比**：必须先在左右孩子中选出较大者，再与父结点比较，否则破坏堆性质。
3. **边界判断**：访问右孩子前要判 `rightIndex <= len`，否则越界。
4. **排序阶段长度递减**：每归位一个元素，未排序区间长度要减 1（代码中 `maxHeapify(0, i - 1)`），已归位的末尾元素不能再参与调整。
5. **升序用大顶堆，降序用小顶堆**——别记反。
6. **稳定性**：堆排序不稳定，对"需要保持相等元素原始顺序"的场景不适用。

## 九、与其他排序的对比

| 排序 | 平均时间 | 最坏时间 | 空间 | 稳定性 |
|------|----------|----------|------|--------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(1) | 不稳定 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 稳定 |
| **堆排序** | **O(n log n)** | **O(n log n)** | **O(1)** | **不稳定** |

**堆排序的优势**：最坏情况仍是 `O(n log n)`（快排最坏会退化到 `O(n²)`），且 **原地排序**、空间 `O(1)`（优于归并的 `O(n)`）。常用于对最坏性能有要求、又不能用额外空间的场景；缺点是常数因子较大、缓存不友好，实践中平均速度通常不及快排。
