# Claude Code 常用命令介绍

> Claude Code 是 Anthropic 官方的命令行 AI 编程助手。本文档介绍日常使用中最常用的命令、快捷键和技巧。

---

## 一、启动与基础

| 命令 | 说明 |
| --- | --- |
| `claude` | 在当前目录启动交互式会话 |
| `claude "你的问题"` | 启动并直接带上首条提问 |
| `claude -c` / `claude --continue` | 继续上一次会话 |
| `claude -r` / `claude --resume` | 选择并恢复某个历史会话 |
| `claude -p "提问"` | 一次性（print 模式）执行后退出，适合脚本/管道 |
| `claude update` | 更新到最新版本 |
| `claude doctor` | 检查安装和环境健康状况 |
| `claude mcp` | 管理 MCP 服务器 |

---

## 二、会话内斜杠命令（Slash Commands）

在会话中输入 `/` 即可触发，常用的有：

| 命令 | 说明 |
| --- | --- |
| `/help` | 查看帮助和所有可用命令 |
| `/clear` | 清空当前对话上下文（开启新话题时常用） |
| `/compact` | 压缩上下文，保留要点、释放空间 |
| `/config` | 打开配置界面（主题、模型等） |
| `/model` | 切换使用的模型（如 Opus / Sonnet / Haiku） |
| `/init` | 为当前代码库生成 `CLAUDE.md` 项目说明文件 |
| `/review` | 审查一个 Pull Request |
| `/cost` | 查看本次会话的 token 用量与费用 |
| `/memory` | 编辑长期记忆文件 |
| `/permissions` | 管理工具权限（允许/拒绝某些操作） |
| `/agents` | 管理子代理（subagents） |
| `/login` `/logout` | 登录 / 登出账号 |
| `/vim` | 开启 Vim 编辑模式 |
| `/terminal-setup` | 配置终端以支持换行等快捷键 |

> 提示：自定义命令（skills）也会以 `/命令名` 形式出现，输入 `/` 时会列出当前可用的全部命令。

## 三、常用快捷键与输入技巧

| 输入 | 作用 |
| --- | --- |
| `! <命令>` | 直接在会话中执行 shell 命令，输出进入对话（如 `! git status`） |
| `@文件路径` | 引用文件，把文件内容带入上下文（支持自动补全） |
| `#内容` | 快速写入记忆 / `CLAUDE.md` |
| `Esc` | 中断当前正在进行的操作 |
| `Esc` `Esc` | 回退/编辑上一条消息 |
| `Shift + Tab` | 切换权限模式（如自动接受编辑、计划模式） |
| `Ctrl + C` | 取消当前输入 |
| `Ctrl + D` | 退出会话 |
| `↑` / `↓` | 浏览历史输入 |
| 拖入图片 | 让 Claude 读取截图/图片（也可粘贴） |

---

## 四、权限模式

通过 `Shift + Tab` 在会话中循环切换：

- **普通模式**：每个写操作/命令都会询问。
- **Plan 模式（计划模式）**：只读分析、先给出方案，不直接改代码。适合先对齐思路。
- **Auto-accept edits（自动接受编辑）**：自动应用文件编辑，减少打断。

## 五、典型使用场景

```bash
# 1. 进入项目，让它先了解代码库
claude
> /init

# 2. 让它修复一个 bug（会自动读文件、改代码、跑测试）
> 修复登录接口返回 500 的问题

# 3. 引用具体文件提问
> @src/com/hm/thread/PrintABC.java 这段交替打印有没有线程安全问题？

# 4. 审查 PR
> /review 123

# 5. 查看花费后清空上下文，开始新任务
> /cost
> /clear
```

---

## 六、实用技巧

1. **多任务并行**：让 Claude 一次性描述完整需求，它会自行拆解、读文件、改代码、验证。
2. **善用 `/clear`**：切换到无关的新任务时清空上下文，能让回答更准、更省 token。
3. **`CLAUDE.md`**：把项目约定、构建命令、代码风格写进去，每次会话自动加载。
4. **`! 命令` 用于交互式登录**：例如 `! gcloud auth login` 这种需要你本人操作的命令，让输出直接进入会话。
5. **计划模式先行**：对复杂改动，先用 Plan 模式确认方案再执行，避免返工。
