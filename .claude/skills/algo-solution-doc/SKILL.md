# 算法解题思路文档（algo-solution-doc）

当你在本仓库**给出、编写或重写任何算法 / LeetCode 题解**时，除了源代码外，必须**额外生成一份中文「解题思路」Markdown 文档**，与算法源文件放在同一目录。

## 触发时机

只要本次任务涉及以下任意一种，就生成文档：
- 编写新的算法 / 题解 Java 文件
- 重写、优化已有算法
- 对算法给出多种解法

## 文档存放位置与命名

- 与算法源文件**同目录**。
- 文件名用中文 + `.md`，体现题号或题名，例如：`回文数解题思路.md`、`两数之和解题思路.md`。
- 若该目录已有同题文档，**更新它而不是新建**。

## 文档结构（必须包含以下小节）

1. **题目描述** — 题意 + 示例表格（输入/输出/说明）+ 原题链接（若有）。
2. **核心观察 / 关键点** — 解题前要抓住的性质（边界、特例、本质转化）。
3. **解法** — 每种解法都要给出：
   - 思路（分步骤文字说明）
   - 关键代码片段
   - 时间复杂度、空间复杂度
   - 优点 / 缺点
4. **解法对比表**（给出多种解法时）— 思路 / 时间 / 空间 / 优缺点一览。
5. **易错点总结** — 边界条件、溢出、奇偶等常见坑。
6. **关联代码** — 指明对应的源文件名。

## 风格要求

- 全程中文讲解，复杂度用 `O(...)` 表示。
- 复杂度优先级：正确性 > 可读性，再谈性能。
- 代码片段与仓库实际源文件保持一致，不要贴伪代码冒充实现。
- 验证算法：写完源码后尽量编译运行测试用例，确认输出正确再写文档结论。
